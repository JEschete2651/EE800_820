\labheader{4-D}{Support Vector Machines with Grid Search and Permutation Importance}

\section{Objective}
Train an \texttt{SVC} classifier (with both linear and \mbox{RBF} kernels) for each of the four tasks, grid-search its hyperparameters over $C$ and $\gamma$, and produce a permutation-importance analysis that gives a \emph{model-agnostic} ranking of which features the classifier relied on. Permutation importance is trustworthy where Gini importance (from LH4-C) can be misleading for correlated features.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handout~4-B complete (\texttt{Code/AIML/artifacts/\{sf,mod,beacon,pkt\}/} with scaled feature matrices). \mbox{SVM}s \emph{require} scaling --- do not attempt to train on unscaled features.
	\item Lab Handout~4-C complete (\texttt{baselines\_manifest.json} and per-task \texttt{rf.pkl} available for comparison in Step~4).
\end{itemize}

\section{Background: Why Both Kernels?}
The linear kernel gives you interpretable coefficients (one per feature per one-vs-rest binary problem) but can only learn linearly separable decision surfaces. The \mbox{RBF} kernel handles non-linear class boundaries at the cost of interpretability and $O(n^2)$ training time. Comparing the two tells you whether the classes are linearly separable in the chosen feature space:

\begin{itemize}[leftmargin=2em]
	\item \textbf{Linear $\approx$ RBF accuracy:} the feature space is (nearly) linearly separable; a linear model is the right tool.
	\item \textbf{RBF $\gg$ Linear:} the decision boundary is curved in this feature space; expect a neural network (LH4-E) to also do well.
	\item \textbf{Linear $>$ RBF:} the \mbox{RBF} is overfitting on noise (rare, usually indicates a bad $\gamma$).
\end{itemize}

\section{Procedure}

\subsection{Step 1 --- Logging, loader, and grid search}
The script (\texttt{Code/AIML/svm\_train.py}) follows the same logging pattern as LH4-A through LH4-C: stdout plus a timestamped archive log and \texttt{svm\_train\_latest.log} in \texttt{Code/AIML/Logs/}. Single-class tasks (\texttt{mod} in the current LoRa-only dataset) are skipped because \texttt{SVC.fit} requires $\geq 2$ classes.

\begin{lstlisting}[language=Python, caption={Top of \texttt{svm\_train.py} --- imports, paths, logging, loader, Wilson CI.}]

from pathlib import Path

from sklearn.model_selection import GridSearchCV

SCRIPT_DIR    = Path(__file__).parent
LOG_DIR       = SCRIPT_DIR / "Logs"
ARTIFACTS_DIR = SCRIPT_DIR / "artifacts"
FIGS_DIR      = SCRIPT_DIR / "figs"

for h in (logging.StreamHandler(sys.stdout),

def load_task(task):
    d = ARTIFACTS_DIR / task
    X_tr = np.load(d / "X_train.npy"); y_tr = np.load(d / "y_train.npy")
    X_va = np.load(d / "X_val.npy");   y_va = np.load(d / "y_val.npy")
    X_te = np.load(d / "X_test.npy");  y_te = np.load(d / "y_test.npy")
    pre  = joblib.load(d / "preprocessor.pkl")
    return X_tr, y_tr, X_va, y_va, X_te, y_te, pre


def wilson_ci(n_correct, n_total, alpha=0.05):
    if n_total == 0:
        return 0.0, 1.0
    p = n_correct / n_total
    z = norm.ppf(1 - alpha / 2)
    denom = 1 + z * z / n_total
    center = (p + z * z / (2 * n_total)) / denom
    half = z * sqrt(p * (1 - p) / n_total
                    + z * z / (4 * n_total * n_total)) / denom
    return max(0.0, center - half), min(1.0, center + half)
\end{lstlisting}

\begin{lstlisting}[language=Python, caption={SVM grid search per task with single-class skip and elapsed-time logging.}]
SVM_GRID = [
    {"kernel": ["linear"], "C": [0.1, 1, 10]},
    {"kernel": ["rbf"],    "C": [0.1, 1, 10, 100],
                           "gamma": ["scale", 0.01, 0.1, 1.0]},
]

results = {}
for task in ["sf", "mod", "beacon", "pkt"]:
    log.info(f"--- Task: {task} ---")
    X_tr, y_tr, X_va, y_va, X_te, y_te, pre = load_task(task)
    n_classes = len(np.unique(y_tr))
    log.info(f"  Train shape={X_tr.shape}, classes={n_classes}, "
             f"features={pre['feature_cols']}")
    if n_classes < 2:
        log.warning(f"  {task}: only {n_classes} class --- skipping (SVC needs >=2)")
        results[task] = {"skipped": True, "reason": "single class in train"}
        continue

    X_cv = np.concatenate([X_tr, X_va]); y_cv = np.concatenate([y_tr, y_va])
    t0 = time.time()
    gs = GridSearchCV(SVC(random_state=42),
                      SVM_GRID, cv=5, scoring="accuracy",
                      n_jobs=-1, verbose=0)
    gs.fit(X_cv, y_cv)
    elapsed = time.time() - t0
    log.info(f"  Grid search complete in {elapsed:.1f}s; "
             f"best params: {gs.best_params_}")

    best = gs.best_estimator_
    y_pred = best.predict(X_te)
    n_correct = int((y_pred == y_te).sum())
    acc_te = n_correct / len(y_te)
    lo, hi = wilson_ci(n_correct, len(y_te))
    log.info(f"  CV acc = {gs.best_score_:.3f}  test acc = {acc_te:.3f}  "
             f"[95% CI {lo:.3f}, {hi:.3f}]")

    joblib.dump({"model": best, "cv_acc": float(gs.best_score_),
                 "test_acc": acc_te, "ci": [lo, hi],
                 "params": gs.best_params_, "elapsed_s": elapsed},
                ARTIFACTS_DIR / task / "svm.pkl")
    results[task] = {
        "kernel":   gs.best_params_.get("kernel"),
        "C":        gs.best_params_.get("C"),
        "gamma":    gs.best_params_.get("gamma"),
        "cv_acc":   float(gs.best_score_),
        "test_acc": acc_te, "ci_lo": lo, "ci_hi": hi,
        "n_correct": n_correct, "n_total": len(y_te),
        "elapsed_s": elapsed,
        "feature_cols": pre["feature_cols"],
    }
\end{lstlisting}

\begin{tcolorbox}[warning={\mbox{SVM} Training Time}]
	At $n \approx 2{,}000$ samples with the full grid (linear: 3 combos; \mbox{RBF}: 16 combos; both with 5-fold \mbox{CV}), expect roughly 30\,s--3\,min per task on a laptop with \texttt{n\_jobs=-1}. If one combination hangs for $>$30 minutes, interrupt it --- $\gamma = 1.0$ with $C = 100$ on a scaled feature set can produce an extremely slow fit due to a near-singular kernel matrix. Remove that corner from the grid and rerun. The script logs per-task elapsed time so this is easy to spot.
\end{tcolorbox}

\subsection{Step 3 --- Permutation importance}
Gini importance from a forest can attribute all the signal to whichever of two correlated features the tree split on first. Permutation importance is model-agnostic and directly measures ``how much does test accuracy drop if I shuffle this column?''

\begin{lstlisting}[language=Python]
def perm_importance(model, X_te, y_te, feature_names, task, n_repeats=20):
    r = permutation_importance(model, X_te, y_te,
                               n_repeats=n_repeats, random_state=42,
                               n_jobs=-1)
    order = np.argsort(r.importances_mean)[::-1]
    fig, ax = plt.subplots(figsize=(7, 3.5))
    ax.boxplot(r.importances[order].T,
               tick_labels=[feature_names[i] for i in order])
    plt.setp(ax.get_xticklabels(), rotation=30, ha="right")
    ax.set_ylabel("accuracy drop when shuffled")
    ax.set_title(f"Permutation importance --- SVM --- {task}")
    fig.tight_layout()
    out = FIGS_DIR / f"perm_importance_svm_{task}.png"
    fig.savefig(out, dpi=150)
    plt.close(fig)
    log.info(f"  Saved {out}")
    ranking = []
    for i in order:
        log.info(f"    {feature_names[i]:20s}  "
                 f"mean={r.importances_mean[i]:+.3f}  "
                 f"std={r.importances_std[i]:.3f}")
        ranking.append((feature_names[i], float(r.importances_mean[i]),
                        float(r.importances_std[i])))
    return ranking


for task, res in results.items():
    if res.get("skipped"):
        continue
    pkl = joblib.load(ARTIFACTS_DIR / task / "svm.pkl")
    _, _, _, _, X_te, y_te, pre = load_task(task)
    log.info(f"--- {task} permutation importance (SVM) ---")
    res["perm_importance"] = perm_importance(
        pkl["model"], X_te, y_te, pre["feature_cols"], task)
\end{lstlisting}

\subsection{Step 4 --- Cross-check against random forest}
Rank the features by permutation importance (SVM, Step~3) and by Gini importance (RF, from \texttt{baselines\_manifest.json}). Compute Spearman rank correlation between the two rankings: $\rho > 0.8$ is strong agreement, $\rho < 0.5$ is informative disagreement (typically caused by correlated features that the two models split differently).

\begin{lstlisting}[language=Python]
baseline_manifest = SCRIPT_DIR / "baselines_manifest.json"
if baseline_manifest.exists():
    with open(baseline_manifest) as f:
        rf_manifest = json.load(f)["results"]
else:
    log.warning("baselines_manifest.json not found --- "
                "skipping RF cross-check (run LH4-C first)")
    rf_manifest = {}

for task, res in results.items():
    if res.get("skipped") or task not in rf_manifest \
            or rf_manifest[task].get("skipped"):
        continue
    rf_rank = [(name, gini) for (name, gini)
               in rf_manifest[task]["feature_importance"]]
    svm_rank = res.get("perm_importance", [])
    common = [c for c, _, _ in svm_rank
              if c in dict(rf_rank)]
    if len(common) < 2:
        log.warning(f"  {task}: too few shared features for rank correlation")
        continue
    rf_ord  = {c: r for r, (c, _) in enumerate(rf_rank)  if c in common}
    svm_ord = {c: r for r, (c, _, _) in enumerate(svm_rank) if c in common}
    rho, _ = spearmanr([rf_ord[c]  for c in common],
                       [svm_ord[c] for c in common])
    log.info(f"  {task}: RF Gini vs SVM perm rank correlation rho = {rho:+.2f}")
    res["rank_correlation_vs_rf"] = float(rho)
\end{lstlisting}

\begin{tcolorbox}[infobox={When the Linear Kernel Wins}]
	If the linear \mbox{SVM} beats the \mbox{RBF} by more than 1 percentage point on a given task, that is a genuinely useful result: it means (a) a simpler, faster model is preferable for deployment, and (b) the problem is linearly separable in the chosen feature space. Do not over-engineer a non-linear solution where a linear one suffices --- the deployment latency budget in LH4-H will thank you.
\end{tcolorbox}
