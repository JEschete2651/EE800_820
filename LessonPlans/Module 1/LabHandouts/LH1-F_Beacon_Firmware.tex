\labheader{1-F}{Cooperative Target Firmware: Bidirectional FSM with PAUSE Listener}

\section{Objective}
Promote the Lab Handout~1-D ping-pong skeleton into production-quality cooperative-target firmware. Both Target~A and Target~B run the same \FSM{} skeleton (\texttt{INIT}$\to$\texttt{CONFIG}$\to$\texttt{IDLE} \\ $\to$\texttt{BUILD}$\to$\texttt{TX}$\to$\texttt{WAIT\_ACK}$\to$\texttt{LOG}$\to$\texttt{IDLE}) with role-specific branches (Target~A initiates DATA on a 1\,s schedule and waits for ACKs; Target~B is normally in \texttt{RX\_LISTEN} and traverses \texttt{BUILD\_ACK}$\to$\texttt{TX\_ACK} on every successful DATA reception). Both firmwares add a shared \texttt{PAUSED} branch entered from any state on receipt of a PAUSE packet from the interceptor (Threat) addressed to the target's own Beacon~ID; while paused, the firmware skips its scheduled DATA transmissions and prints \texttt{"I'm Jammed"} on USART2 once per pause-entry. Validate two-target ping-pong over a 5-minute run and validate the PAUSE behavioral response with the Threat firmware from Lab Handout~1-G.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handouts~1-A, 1-B, 1-C, 1-D complete.
	\item Target~A and Target~B from LH1-D running ping-pong successfully ($\geq$95\% ACK rate at 1\,m).
	\item Lab Handout~1-G (interceptor / Threat firmware) is required for the PAUSE behavioral validation in Step~7. If LH1-G is not yet complete, finish LH1-G first; the two handouts are co-validated.
\end{itemize}

\begin{tcolorbox}[warning={LH1-F, LH1-G, and LH1-H are tightly interconnected --- read this first}]
The three Module~1 firmware/integration handouts cannot be \emph{fully} completed in isolation. Plan to work through all three in alternation; expect to revisit each as the others come online.
\begin{itemize}[leftmargin=1.2em]
	\item \textbf{LH1-F (this handout)} produces the cooperative-target firmware for two boards and the DATA/ACK ping-pong traffic source. The PAUSE listener at the end of LH1-F (Step~7, ``I'm~Jammed'' behavior) cannot be validated until LH1-G is built and transmitting PAUSE packets at the Targets.
	\item \textbf{LH1-G (interceptor / Threat firmware)} consumes the LH1-F ping-pong traffic as its capture source. It cannot be validated until LH1-F Steps~1--6 (DATA/ACK without the PAUSE listener) are running. LH1-G's host-side parser step requires a USB-to-UART adapter; if one is unavailable, LH1-G is considered ``code-complete'' once it builds and the on-board log shows packets being received --- the binary frame stream is then validated by LH1-H.
	\item \textbf{LH1-H (FPGA ingestion)} consumes the LH1-G binary forwarding frames and presents an end-to-end validation that does not require any extra host hardware (the Nexys~A7 seven-segment display acts as a frame counter). It cannot be validated until LH1-G is producing frames on \texttt{USART3 TX}.
\end{itemize}
Recommended pass order: complete LH1-F Steps~1--6 (DATA/ACK only), then build LH1-G end-to-end against that traffic, then return here to finish LH1-F Step~7 (PAUSE listener), and finally close out with LH1-H. The Module~1 deliverable is the union of all three.
\end{tcolorbox}

\section{\FSM{} Specification}
Both Target~A and Target~B run the same shared \FSM{} skeleton. The role-specific differences are confined to the \texttt{IDLE}, \texttt{BUILD}, and \texttt{WAIT\_ACK} state implementations; everything else is identical:
\begin{enumerate}[leftmargin=2em]
	\item \textbf{INIT}: \HAL{} initialization, clock setup, peripheral configuration.
	\item \textbf{CONFIG}: reset SX1276, verify \texttt{RegVersion}, program all LoRa registers per Table~\ref{tab:lora-params-f}, put modem in \texttt{STANDBY}, install the \texttt{DIO0} \texttt{EXTI} ISR.
	\item \textbf{IDLE}: Target~A: poll \texttt{ms\_count} against \texttt{next\_tx\_ms}; transition to \texttt{BUILD\_DATA} when due. Target~B: enter \texttt{RXCONTINUOUS}; remain in \texttt{IDLE} until the ISR sets a \texttt{rx\_ready} flag; transition to either \texttt{BUILD\_ACK} (if the received packet is a DATA from a peer Beacon~ID) or back to \texttt{IDLE} (if the packet is anything else --- PAUSE addressed to a different ID, etc.). Both: at every \texttt{IDLE} entry, check \texttt{paused\_until\_ms}; if past expiry, clear the paused flag.
	\item \textbf{BUILD\_DATA} (Target~A): populate the 32-byte packet with \texttt{pkt\_type=DATA}, \texttt{mod\_kind=LoRa}, current \texttt{seq}, and a structured payload (see Step~2 below). Compute \CRC{}-16. Increment \texttt{seq}.
	\item \textbf{BUILD\_ACK} (Target~B): populate the 32-byte packet with \texttt{pkt\_type=ACK}, \texttt{mod\_kind=LoRa}, the received DATA's \texttt{seq} echoed into the sequence field, and \texttt{0xAA} payload. Compute \CRC{}-16.
	\item \textbf{TX}: write \texttt{FIFO}, command \texttt{TX} mode, wait for \texttt{TxDone} with a 500\,ms timeout. On timeout, transition to \texttt{FAULT}.
	\item \textbf{WAIT\_ACK} (Target~A only): switch radio into \texttt{RXCONTINUOUS}; wait up to 200\,ms for an ACK from Target~B with matching \texttt{seq}. On match, capture the round-trip time. On timeout, log a missed-ACK event and continue.
	\item \textbf{LOG}: \texttt{printf} a single-line status (CSV format consistent with LH1-D) over USART2 and return to \texttt{IDLE}.
	\item \textbf{PAUSED} (entered by the \texttt{DIO0} ISR or main-loop poll on receipt of a PAUSE addressed to this target's ID): set \texttt{paused\_until\_ms = HAL\_GetTick() + (pause\_payload[1] $\times$ 100)}. Print \texttt{"I'm Jammed (by ID=0xFF, until t=\%lu)"} on USART2 \emph{only on the first entry per pause window} (throttled, not per-tick). On subsequent ISR PAUSE refreshes within the window, only update \texttt{paused\_until\_ms}; do not reprint. Skip all DATA TX (Target~A) while paused. ACK TX (Target~B) is permitted because it is a response to a real DATA reception. Return to \texttt{IDLE} on next main-loop iteration.
	\item \textbf{FAULT}: \texttt{printf} an error line over USART2; clear all radio IRQ flags; force \texttt{CONFIG} state.
\end{enumerate}

\begin{lstlisting}[language=C, caption={\texttt{Core/Src/lh1f\_packet.c} --- create this new file in both projects. Implements the CRC and the canonical packet builder used by every \texttt{BUILD\_*} state.}]
#include "lh1f_defs.h"
#include "main.h"

extern volatile uint32_t ms_count;

/* CRC-16/CCITT-FALSE: poly=0x1021, init=0xFFFF, no reflect, xorout=0x0000.
 * Verified against the standard test vector: crc16_ccitt("123456789", 9)
 * must equal 0x29B1.
 */
uint16_t crc16_ccitt(const uint8_t *buf, uint32_t len) {
    uint16_t crc = 0xFFFFu;
    for (uint32_t i = 0; i < len; i++) {
        crc ^= ((uint16_t)buf[i]) << 8;
        for (int b = 0; b < 8; b++) {
            crc = (crc & 0x8000u) ? (uint16_t)((crc << 1) ^ 0x1021u)
                                  : (uint16_t)(crc << 1);
        }
    }
    return crc;
}

/* build_packet: populate the canonical 32-byte radio packet.
 * Caller is responsible for any payload customization AFTER this call
 * (e.g. PAUSE adds target-id/hold-off in bytes 13-14 then RECOMPUTES CRC).
 */
void build_packet(uint8_t out[32], uint8_t my_id, uint16_t seq,
                  uint8_t pkt_type, uint8_t mod_kind)
{
    /* Preamble */
    out[0] = 0xAEu;
    out[1] = 0x5Au;
    /* Sender + sequence */
    out[2] = my_id;
    out[3] = (uint8_t)(seq & 0xFFu);
    out[4] = (uint8_t)((seq >> 8) & 0xFFu);
    /* TX power */
    out[5] = 14u;
    /* Packed Modulation Code byte: bits[7:2] = mod_kind << 2,
     *                              bits[1:0] = pkt_type            */
    out[6] = (uint8_t)((mod_kind << 2) | (pkt_type & 0x03u));
    /* Spreading factor / bandwidth code */
    out[7] = (mod_kind == MOD_KIND_LORA) ? 7u : 0u;   /* SF7 default */
    out[8] = 0u;                                       /* 125 kHz BW  */
    /* Timestamp */
    uint32_t t = ms_count;
    out[9]  = (uint8_t)(t      );
    out[10] = (uint8_t)(t >>  8);
    out[11] = (uint8_t)(t >> 16);
    out[12] = (uint8_t)(t >> 24);
    /* Default payload: 0xAA filler. Caller may overwrite bytes 13-29
     * before transmit (e.g. for a PAUSE packet's target-id field).    */
    for (int i = 13; i < 30; i++) out[i] = 0xAAu;
    /* CRC-16/CCITT-FALSE over bytes 0-29, MSB first */
    uint16_t crc = crc16_ccitt(out, 30);
    out[30] = (uint8_t)(crc >> 8);
    out[31] = (uint8_t)(crc);
}
\end{lstlisting}

\section{Procedure}

\subsection{Step 1 --- Open the existing TargetA and TargetB projects}

The LH1-D \texttt{Nucleo\_RFM95W\_TargetA} and \texttt{Nucleo\_RFM95W\_TargetB} projects continue to be the source-of-truth for the cooperative-target firmware. \textbf{Do not copy, rename, or move them, and do not run CubeMX again.} The LH1-F additions extend the LH1-D code with a proper \FSM{}, the PAUSE listener path, and the DIO0 \texttt{EXTI} interrupt handler --- all of these are pure C-level edits inside the existing USER~CODE blocks. The CubeMX peripheral configuration was already done in LH1-C Step~1 (SPI1, USART2, USART3 with DMA, TIM2 1\,ms tick, \texttt{RFM\_CS}/\texttt{RFM\_RST}/\texttt{RFM\_DIO0} GPIO labels with EXTI on PA8 and the EXTI[9:5] NVIC line enabled). No re-generation is required.

\begin{enumerate}[leftmargin=2em]
	\item Open both \texttt{Nucleo\_RFM95W\_TargetA} and \texttt{Nucleo\_RFM95W\_TargetB} in separate VS Code windows (same setup as LH1-D).
	\item Press \texttt{F7} in each window to confirm both projects still build with the LH1-D code in place.
\end{enumerate}

\subsection{Step 2 --- Wire the TIM2 1\,ms tick into the C code}

CubeMX (LH1-C Step~1.7) already enabled \texttt{TIM2} as a 1\,ms periodic timer and enabled its NVIC interrupt; the C-side \texttt{ms\_count} variable and the \texttt{HAL\_TIM\_PeriodElapsedCallback} override still need to be added.

\begin{enumerate}[leftmargin=2em]
	\item In both \texttt{TargetA} and \texttt{TargetB} \texttt{main.c} files, add to \texttt{/* USER CODE BEGIN PV */}:
\begin{verbatim}
/* USER CODE BEGIN PV */
volatile uint32_t ms_count = 0;
/* USER CODE END PV */
\end{verbatim}
	\item Add the callback in \texttt{/* USER CODE BEGIN 4 */}:
\begin{verbatim}
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM2) ms_count++;
}
\end{verbatim}
	\item Inside \texttt{/* USER CODE BEGIN 2 */} (after \texttt{lora\_init()}), start the timer in interrupt mode:
\begin{verbatim}
HAL_TIM_Base_Start_IT(&htim2);
\end{verbatim}
	\item In Target~A only, schedule the DATA cadence: store \texttt{static uint32\_t next\_tx\_ms} in the main loop and advance it by 1000\,ms after each successful \texttt{BUILD\_DATA}$\to$\texttt{TX}$\to$\texttt{WAIT\_ACK}$\to$\texttt{LOG} cycle. Target~B does not schedule transmissions; it only responds.
\end{enumerate}

\subsection{Step 3 --- DIO0 EXTI ISR and shared receive path}

Both targets handle the SX1276's \texttt{RxDone} interrupt via the \texttt{DIO0} pin's \texttt{EXTI} line. Implement the ISR in \texttt{/* USER CODE BEGIN 4 */}:
\begin{verbatim}
typedef struct {
    uint8_t  raw[32];
    uint32_t rx_ts_ms;
    int8_t   rssi_dbm;
    int8_t   snr_q025;
} pkt_rx_t;

volatile pkt_rx_t  rx_slot;
volatile uint8_t   rx_ready = 0;
volatile uint32_t  paused_until_ms = 0;
volatile uint32_t  pause_print_until_ms = 0;  /* throttle "I'm Jammed" */

void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    if (GPIO_Pin != RFM_DIO0_Pin) return;

    uint8_t irq = rfm_read_reg(0x12);
    if ((irq & 0x40) == 0) {
        rfm_write_reg(0x12, 0xFF);
        return;
    }
    if (irq & 0x20) {  /* CRC error */
        rfm_write_reg(0x12, 0xFF);
        return;
    }

    /* Read the packet into rx_slot */
    uint8_t rx_addr = rfm_read_reg(0x10);
    rfm_write_reg(0x0D, rx_addr);
    for (int i = 0; i < 32; i++) {
        ((uint8_t*)rx_slot.raw)[i] = rfm_read_reg(0x00);
    }
    rx_slot.rx_ts_ms = ms_count;
    rx_slot.rssi_dbm = (int8_t)((int16_t)rfm_read_reg(0x1A) - 157);
    rx_slot.snr_q025 = (int8_t)rfm_read_reg(0x19);
    rfm_write_reg(0x12, 0xFF);

    /* Check for PAUSE addressed to us --- handled in ISR for low latency */
    uint8_t sender   = rx_slot.raw[2];
    uint8_t pkt_type = rx_slot.raw[6] & 0x03;
    if (pkt_type == PKT_TYPE_PAUSE
        && sender == BEACON_ID_THREAT
        && rx_slot.raw[13] == MY_BEACON_ID) {
        uint8_t units = rx_slot.raw[14];
        paused_until_ms = ms_count + ((uint32_t)units * 100);
        /* Defer printf to main loop to avoid blocking ISR */
        rx_ready = 1;  /* main loop notices the PAUSE and prints once */
        return;
    }

    rx_ready = 1;  /* main loop will dispatch DATA / ACK */
}
\end{verbatim}
The constant \texttt{MY\_BEACON\_ID} is defined per project: Target~A defines \texttt{\#define MY\_BEACON\_ID BEACON\_ID\_A}, Target~B defines \texttt{\#define MY\_BEACON\_ID BEACON\_ID\_B}.

\subsection{Step 4 --- Main-loop \FSM{} skeleton}

In \texttt{/* USER CODE BEGIN WHILE */}, replace the LH1-D inline-loop logic with the explicit \FSM{}:
\begin{verbatim}
typedef enum {
    ST_IDLE, ST_BUILD_DATA, ST_BUILD_ACK, ST_TX,
    ST_WAIT_ACK, ST_LOG, ST_FAULT
} state_t;

state_t state = ST_IDLE;
uint16_t seq = 0;
uint16_t pending_ack_seq = 0;
uint32_t t_tx_ms = 0;
uint8_t  pkt[32];
uint8_t  log_was_acked = 0;
int16_t  log_ack_rssi = 0;
uint32_t log_rtt_ms = 0;

while (1) {
    /* Honor PAUSE: if paused, suppress DATA scheduling but still
       process incoming RX (so ACKs can still be sent and PAUSE
       refreshes can extend the hold-off). */
    uint8_t paused_now = (ms_count < paused_until_ms);

    /* Throttled "I'm Jammed" print: emit at most once per pause entry */
    if (paused_now && pause_print_until_ms < paused_until_ms) {
        printf("I'm Jammed (by ID=0x%02X, until t=%lu)\r\n",
               BEACON_ID_THREAT, (unsigned long)paused_until_ms);
        pause_print_until_ms = paused_until_ms;
    }

    switch (state) {
    case ST_IDLE:
        /* Target A initiator path */
#ifdef ROLE_INITIATOR
        if (!paused_now && ms_count >= next_tx_ms) {
            state = ST_BUILD_DATA;
            break;
        }
#endif
        /* Both: handle any received packet */
        if (rx_ready) {
            rx_ready = 0;
            uint8_t pkt_type = rx_slot.raw[6] & 0x03;
            uint8_t sender   = rx_slot.raw[2];

#ifdef ROLE_RESPONDER
            if (pkt_type == PKT_TYPE_DATA && sender == BEACON_ID_A) {
                pending_ack_seq = (uint16_t)rx_slot.raw[3]
                                | ((uint16_t)rx_slot.raw[4] << 8);
                state = ST_BUILD_ACK;
                break;
            }
#endif
            /* Target A waiting for an ACK while in IDLE means the
               WAIT_ACK timeout already expired; ignore stray ACKs. */
            (void)pkt_type; (void)sender;
        }
        break;

    case ST_BUILD_DATA:
        build_packet(pkt, MY_BEACON_ID, seq,
                     PKT_TYPE_DATA, MOD_KIND_LORA);
        state = ST_TX;
        break;

    case ST_BUILD_ACK:
        build_packet(pkt, MY_BEACON_ID, pending_ack_seq,
                     PKT_TYPE_ACK, MOD_KIND_LORA);
        state = ST_TX;
        break;

    case ST_TX: {
        rfm_set_mode_standby();
        rfm_write_reg(0x0D, rfm_read_reg(0x0E));
        for (int i = 0; i < 32; i++) rfm_write_reg(0x00, pkt[i]);
        rfm_write_reg(0x22, 32);
        t_tx_ms = ms_count;
        rfm_write_reg(0x12, 0xFF);
        rfm_write_reg(0x01, 0x83);  /* TX */

        uint32_t tx_deadline = ms_count + 500;
        uint8_t ok = 0;
        while (ms_count < tx_deadline) {
            if (rfm_read_reg(0x12) & 0x08) { ok = 1; break; }
        }
        rfm_write_reg(0x12, 0xFF);
        if (!ok) { state = ST_FAULT; break; }

#ifdef ROLE_INITIATOR
        rfm_set_mode_rx();
        state = ST_WAIT_ACK;
#else
        rfm_set_mode_rx();
        state = ST_LOG;
#endif
        break;
    }

#ifdef ROLE_INITIATOR
    case ST_WAIT_ACK: {
        uint32_t deadline = ms_count + 200;
        log_was_acked = 0;
        while (ms_count < deadline) {
            if (rx_ready) {
                rx_ready = 0;
                uint8_t pkt_type = rx_slot.raw[6] & 0x03;
                uint8_t sender   = rx_slot.raw[2];
                uint16_t rs = (uint16_t)rx_slot.raw[3]
                            | ((uint16_t)rx_slot.raw[4] << 8);
                if (pkt_type == PKT_TYPE_ACK
                    && sender == BEACON_ID_B
                    && rs == seq) {
                    log_was_acked = 1;
                    log_ack_rssi  = rx_slot.rssi_dbm;
                    log_rtt_ms    = ms_count - t_tx_ms;
                    break;
                }
            }
        }
        next_tx_ms = t_tx_ms + 1000;  /* schedule next on a fixed grid */
        seq++;
        state = ST_LOG;
        break;
    }
#endif

    case ST_LOG:
#ifdef ROLE_INITIATOR
        if (log_was_acked) {
            printf("CSV,%u,%lu,%lu,%lu,%d\r\n",
                   (unsigned)(seq - 1), (unsigned long)t_tx_ms,
                   (unsigned long)(t_tx_ms + log_rtt_ms),
                   (unsigned long)log_rtt_ms, (int)log_ack_rssi);
        } else {
            printf("CSV,%u,%lu,0,0,0  (no ACK)\r\n",
                   (unsigned)(seq - 1), (unsigned long)t_tx_ms);
        }
#else
        printf("ACK seq=%u rssi=%d\r\n",
               (unsigned)pending_ack_seq, (int)rx_slot.rssi_dbm);
#endif
        state = ST_IDLE;
        break;

    case ST_FAULT:
        printf("FAULT --- TX timeout, reconfiguring radio\r\n");
        rfm_write_reg(0x12, 0xFF);
        lora_init();
        rfm_set_mode_rx();
        state = ST_IDLE;
        break;
    }
}
\end{verbatim}

\subsection{Step 5 --- Define the role and Beacon~ID per project}

The same \FSM{} source above compiles into either Target~A or Target~B firmware via two compile-time defines per project. Because LH1-C built both projects with the VS Code STM32 extension's CMake flow, set the defines in \texttt{CMakeLists.txt} (do not re-run CubeMX). Open each project's \texttt{CMakeLists.txt}, find the existing \texttt{target\_compile\_definitions(\$\{CMAKE\_PROJECT\_NAME\} PRIVATE ...)} call (CubeMX generates one with \texttt{USE\_HAL\_DRIVER} and \texttt{STM32L476xx}) and append the two role lines.

\textbf{Target~A project --- in \texttt{Nucleo\_RFM95W\_TargetA/CMakeLists.txt}}:
\begin{lstlisting}[language=CMake, caption={Append to the existing \texttt{target\_compile\_definitions} call.}]
target_compile_definitions(${CMAKE_PROJECT_NAME} PRIVATE
    USE_HAL_DRIVER
    STM32L476xx
    ROLE_INITIATOR
    MY_BEACON_ID=BEACON_ID_A
)
\end{lstlisting}

\textbf{Target~B project --- in \texttt{Nucleo\_RFM95W\_TargetB/CMakeLists.txt}}:

The \texttt{\#ifdef ROLE\_INITIATOR} / \texttt{\#ifdef ROLE\_RESPONDER} guards in the \FSM{} above then compile each project into its role.

\subsection{Step 6 --- GPIO instrumentation pulse}
Add a \mbox{GPIO} output pulse on the entry to the \texttt{TX} state. This gives a scope trigger for measuring transmission timing in Module~3.

To avoid re-running CubeMX (which can break the project's CMake state), configure the trigger pin entirely from C using direct \HAL{} GPIO calls. \texttt{PB10} is free on the Nucleo-L476RG morpho header (CN10 pin~25) and is not used by any peripheral configured in LH1-C.

\begin{enumerate}[leftmargin=2em]
	\item In \texttt{/* USER CODE BEGIN 2 */}, before the main loop, configure \texttt{PB10} as a push-pull output:
\begin{verbatim}
__HAL_RCC_GPIOB_CLK_ENABLE();
GPIO_InitTypeDef trig = {0};
trig.Pin   = GPIO_PIN_10;
trig.Mode  = GPIO_MODE_OUTPUT_PP;
trig.Pull  = GPIO_NOPULL;
trig.Speed = GPIO_SPEED_FREQ_HIGH;
HAL_GPIO_Init(GPIOB, &trig);
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_10, GPIO_PIN_RESET);
\end{verbatim}
	\item In the \texttt{ST\_TX} case, immediately before commanding \texttt{TX} mode, raise the pin:
\begin{verbatim}
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_10, GPIO_PIN_SET);
\end{verbatim}
	After the \texttt{TxDone} flag clears (i.e., at the bottom of the \texttt{ST\_TX} case before transitioning out), lower it:
\begin{verbatim}
HAL_GPIO_WritePin(GPIOB, GPIO_PIN_10, GPIO_PIN_RESET);
\end{verbatim}
	\item Wire a scope probe to Nucleo CN10 pin~25 (PB10), with ground on any GND pin (e.g., CN10 pin~9). A scope or logic analyzer on this pin should show evenly-spaced pulses (1\,s period for Target~A) when the firmware is not paused, and silence on Target~A while the Threat's user button is held.
\end{enumerate}
