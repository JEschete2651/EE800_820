\labheader{1-G}{Interceptor (Threat) Firmware: ISR-Driven Capture, RSSI Table, and Button-Gated PAUSE TX}

\section{Objective}
Implement the third radio's firmware: the interceptor (``Threat'', Beacon~ID \texttt{0xFF}) that passively captures every DATA, ACK, and PAUSE frame on the shared 906.5\,MHz channel via an \texttt{RxDone} interrupt, queues frames in a ring buffer, and forwards them from the main loop to the \FPGA{} over a 43-byte framed UART protocol. In parallel, the firmware maintains a per-Beacon-ID \RSSI{} observation table that is updated on every received packet, and runs a button-gated PAUSE TX scheduler: while the user button (B1) on the Threat board is held, the firmware emits a PAUSE packet at $\sim$100\,ms cadence addressed to whichever non-stale Beacon~ID currently has the highest \RSSI{}, with a 500\,ms hold-off duration in the payload. This is the software side of the Session~4 deliverable.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handouts~1-A, 1-B, 1-C, 1-D complete.
	\item Lab Handout~1-F (cooperative target firmware) at least partially complete --- the two Targets must be running ping-pong as a traffic source for Threat capture validation. Steps 1--6 of LH1-F (DATA/ACK ping-pong without the PAUSE listener) are the minimum required.
	\item \textbf{Optional} for the Step~7 host-side parser validation: a 3.3\,V USB-to-UART adapter (CP2102, CH340, FT232, etc.). If one is not available, skip Step~7 and validate the forwarding stream end-to-end against the \FPGA{} in Lab Handout~1-H instead --- the same 43-byte frames are consumed by the Nexys~A7, with the seven-segment display acting as a live frame counter.
\end{itemize}

\begin{tcolorbox}[infobox={USART3 is not on the ST-Link USB Port}]
The Nucleo's single USB-B port is the ST-Link Virtual COM Port, which is wired internally to \texttt{USART2} only (\texttt{PA2}/\texttt{PA3}). \texttt{USART3} (\texttt{PC4}/\texttt{PC5} on this project) lands on the \mbox{Morpho} header pins and is \emph{not} routed to that USB port. The Threat will print its \texttt{printf} debug log (boot banner, per-RX rows, \texttt{PAUSE -> ID=...}) on \texttt{USART2} as before, but the 43-byte binary forwarding frames only appear on the \texttt{PC4} pin --- you must either jumper that pin to a USB-UART adapter (Step~7) or to the Nexys~A7 \mbox{PMOD} (LH1-H) before any host or FPGA can see them.
\end{tcolorbox}

\section{Forwarding Frame Format}
The interceptor wraps each received radio packet (DATA, ACK, or PAUSE --- the FPGA does not care which) in a framed UART message. Total 43 bytes. This layout is the canonical spec from the Module~1 lesson plan (Table~6) and is consumed verbatim by the Module~2 parser; do not reorder fields.
\begin{table}[h!]
	\centering
	\begin{tabular}{c l l}
		\toprule
		\textbf{Offset} & \textbf{Field} & \textbf{Length} \\
		\midrule
		0 & Start delimiter (\texttt{0x7E}) & 1 \\
		1 & Frame length (always \texttt{0x29} = 41) & 1 \\
		2--5 & \mbox{RX} timestamp (uint32, \mbox{LE}, ms) & 4 \\
		6 & \RSSI{} (int8, dBm, two's complement) & 1 \\
		7 & \SNR{} (int8, dB, 0.25\,dB resolution) & 1 \\
		8--9 & Frequency error (int16, \mbox{LE}, Hz) & 2 \\
		10--41 & Radio packet (32 bytes: DATA, ACK, or PAUSE) & 32 \\
		42 & Frame \CRC{}-8/ITU (poly \texttt{0x07}) over bytes 1--41 & 1 \\
		\bottomrule
	\end{tabular}
\end{table}

\begin{tcolorbox}[infobox={Length-Field Rule}]
The frame-length byte is defined as the count of bytes \emph{excluding the start delimiter and the trailing \CRC{}}. For a 43-byte frame, byte[1] must equal \texttt{0x29} (41 decimal). The radio packet's embedded \CRC{}-16/CCITT (radio packet bytes 30--31) lands at frame offsets 40--41 and is \emph{not} the frame \CRC{}; the frame \CRC{}-8 is at offset 42.
\end{tcolorbox}

\labfigure{LH1G_frame_layout.png}{0.9\textwidth}{43-byte forwarding frame layout: delimiter, length, metadata (timestamp, \RSSI{}, \SNR{}, freq.\ error), embedded 32-byte radio packet, and trailing \CRC{}-8 at byte 42.}{fig:lh1g-frame}

\section{Architecture}

\subsection{ISR path}
\begin{enumerate}[leftmargin=2em]
	\item DIO0 rising edge triggers an \texttt{EXTI} interrupt.
	\item \ISR{} reads \texttt{RegIrqFlags}. If \texttt{RxDone} is set and \texttt{PayloadCrcError} is clear:
	\begin{enumerate}[leftmargin=1.2em, label=(\alph*)]
		\item Read \texttt{RegFifoRxCurrentAddr}, set \texttt{RegFifoAddrPtr}, read 32 bytes from \texttt{RegFifo}.
		\item Read \texttt{RegPktRssiValue}, \texttt{RegPktSnrValue}, and \texttt{RegFei*} for the frequency error.
		\item Stamp the current \texttt{ms\_count}.
		\item Push a struct (\texttt{intercepted\_pkt\_t}) containing these fields into a \textit{lock-free single-producer/single-consumer ring buffer} (producer = \ISR{}, consumer = main loop).
		\item Update the per-Beacon-ID \RSSI{} table: \texttt{rssi\_table[sender\_id] = \{rssi\_dbm, ms\_count\}}, where \texttt{sender\_id} is the radio packet's byte[2].
		\item Clear \texttt{RegIrqFlags} by writing \texttt{0xFF}.
	\end{enumerate}
\end{enumerate}

\subsection{Main-loop path}
\begin{enumerate}[leftmargin=2em]
	\item Check ring buffer head vs.\ tail. If not empty, dequeue one packet.
	\item Build the 43-byte forwarding frame.
	\item Compute the frame \CRC{}-8/ITU (poly \texttt{0x07}, init \texttt{0x00}) over bytes 1--41 and place it in byte 42. The embedded radio-packet \CRC{}-16 (already in the 32-byte payload) is copied through unchanged at frame offsets 40--41.
	\item Transmit the entire 43-byte frame over \texttt{USART3} (or whichever \UART{} is wired to the Nexys A7) using \texttt{HAL\_UART\_Transmit\_DMA}.
	\item Increment a \texttt{frames\_forwarded} counter.
	\item Poll the user button (B1, \texttt{PC13}, active-low). If pressed, run the PAUSE TX scheduler (Step~5 below).
\end{enumerate}

\labfigure{LH1G_ring_buffer_diagram.png}{0.75\textwidth}{Single-producer/single-consumer ring buffer: \ISR{} advances \texttt{head}, main loop advances \texttt{tail}. A \texttt{\_\_DMB()} barrier orders the payload write against the head increment.}{fig:lh1g-ring}

\subsection{Ring buffer sizing}
A 16-entry ring ($\approx$640 bytes of intercepted-packet structs) accommodates roughly 16 back-to-back receptions without overflow. With Target~A and Target~B in 1\,s ping-pong (effectively two captured frames per second) plus PAUSE bursts at 10\,Hz while the button is held, a 16-entry ring gives $\sim$1.5\,s of margin against a stalled main loop --- comfortable.

\subsection{RSSI observation table}
A 256-entry array indexed by Beacon~ID, with each entry containing the most recent \RSSI{} (in dBm) and the \texttt{ms\_count} at which that observation was captured. The \texttt{last\_seen\_ms} field is what makes the table self-pruning: an entry is considered \emph{stale} (and excluded from highest-\RSSI{} lookups) if \texttt{ms\_count - last\_seen\_ms} exceeds a configurable threshold (5\,s default). Without this discipline, a Target that briefly transmitted hours ago and then went quiet would still appear in the table at its old \RSSI{} value and could become the PAUSE target by mistake --- a silent integration fault that the Module~5 failure-mode catalog records as the canonical depth-2 entry tied to Module~1's staleness threshold choice.

\subsection{PAUSE TX scheduler}
A button-gated, main-loop-driven scheduler. Pseudocode:
\begin{enumerate}[leftmargin=2em]
	\item If \texttt{HAL\_GPIO\_ReadPin(B1\_GPIO\_Port, B1\_Pin) == GPIO\_PIN\_RESET} (button pressed, active-low) and \texttt{ms\_count - last\_pause\_tx\_ms $\geq$ 100}:
	\begin{enumerate}[leftmargin=1.2em, label=(\alph*)]
		\item Scan the \RSSI{} table. Pick the non-stale Beacon~ID with the highest \RSSI{}.
		\item If no non-stale entry exists, log \texttt{"no PAUSE target available"} and skip.
		\item Build a 32-byte PAUSE packet via \texttt{build\_packet(buf, BEACON\_ID\_THREAT, pause\_seq, PKT\_TYPE\_PAUSE, MOD\_KIND\_LORA)}, then overwrite \texttt{buf[13] = chosen\_id} and \texttt{buf[14] = 5} (5 units $\times$ 100\,ms = 500\,ms hold-off), then \emph{recompute} the \CRC{}-16 over bytes 0--29 (the payload bytes changed).
		\item Switch the radio out of \texttt{RXCONTINUOUS} (the receive path is unavoidably interrupted for the $\sim$50\,ms PAUSE airtime; this is the documented half-duplex cost).
		\item TX the PAUSE packet.
		\item Switch back to \texttt{RXCONTINUOUS}.
		\item Log on USART2: \texttt{"PAUSE -> ID=\%02X (RSSI=\%d dBm, seq=\%lu)"}.
		\item \texttt{last\_pause\_tx\_ms = ms\_count}; \texttt{pause\_seq++}.
	\end{enumerate}
\end{enumerate}

\subsection{Step 2 --- Ring buffer and intercepted-packet struct}

Add the ring buffer definitions and the explicit \texttt{push}/\texttt{pop} primitives to \texttt{main.c}. The push runs in \ISR{} context; the pop runs in main-loop context. They are coupled by a single \texttt{\_\_DMB()} barrier on the producer side that orders the payload write before the head increment.

\begin{lstlisting}[language=C, caption={Add inside \texttt{/* USER CODE BEGIN PTD */} (typedef) and \texttt{/* USER CODE BEGIN PV */} (storage).}]
/* --- USER CODE BEGIN PTD --- */
typedef struct {
    uint8_t  raw[32];          /* 32-byte radio packet as received */
    uint32_t rx_timestamp_ms;  /* ms tick at RxDone */
    int8_t   rssi_dbm;
    int8_t   snr_q025;         /* 0.25 dB resolution */
    int16_t  freq_error_hz;
} intercepted_pkt_t;
/* --- USER CODE END PTD --- */

/* --- USER CODE BEGIN PV --- */
#define RING_SIZE 16u                          /* must be power of two */
#define RING_MASK (RING_SIZE - 1u)

static intercepted_pkt_t ring[RING_SIZE];
volatile uint32_t ring_head = 0;               /* writer (ISR)         */
volatile uint32_t ring_tail = 0;               /* reader (main loop)   */
volatile uint32_t dropped   = 0;
/* --- USER CODE END PV --- */
\end{lstlisting}

\begin{lstlisting}[language=C, caption={Place inside \texttt{/* USER CODE BEGIN 4 */}. Single-producer / single-consumer; no critical section needed because head and tail are written by exactly one context each.}]
/* Producer: called from the DIO0 ISR. Returns 1 on success, 0 on overflow. */
static inline int ring_push(const intercepted_pkt_t *src) {
    uint32_t head = ring_head;
    uint32_t next = (head + 1u) & RING_MASK;
    if (next == ring_tail) {
        dropped++;
        return 0;
    }
    ring[head] = *src;     /* full struct copy, ~46 bytes */
    __DMB();               /* order payload write before head update */
    ring_head = next;
    return 1;
}

/* Consumer: called from the main loop. Returns 1 if a packet was dequeued. */
static inline int ring_pop(intercepted_pkt_t *dst) {
    uint32_t tail = ring_tail;
    if (tail == ring_head) return 0;     /* empty */
    *dst = ring[tail];
    __DMB();
    ring_tail = (tail + 1u) & RING_MASK;
    return 1;
}
\end{lstlisting}

\subsection{Step 3 --- RSSI observation table}
\begin{enumerate}[leftmargin=2em]
	\item Define the table:
\begin{verbatim}
typedef struct {
    int8_t   rssi_dbm;
    uint32_t last_seen_ms;
} rssi_obs_t;

#define RSSI_STALE_MS 5000
static volatile rssi_obs_t rssi_table[256];
\end{verbatim}
	\item In \texttt{main()} after \texttt{lora\_init()}, initialize:
\begin{verbatim}
for (int i = 0; i < 256; i++) {
    rssi_table[i].rssi_dbm = -128;
    rssi_table[i].last_seen_ms = 0;
}
\end{verbatim}
	\item Update inside the ISR (after the FIFO read, before the ring \texttt{push}):
\begin{verbatim}
uint8_t sender = pkt_buf.raw[2];
rssi_table[sender].rssi_dbm     = pkt_buf.rssi_dbm;
rssi_table[sender].last_seen_ms = pkt_buf.rx_timestamp_ms;
\end{verbatim}
	\item Implement the highest-RSSI lookup as a main-loop helper:
\begin{verbatim}
int find_highest_rssi_id(uint8_t *out_id) {
    int8_t best_rssi = -128;
    int best_id = -1;
    uint32_t now = ms_count;
    for (int i = 1; i < 256; i++) {  /* skip 0x00 = invalid */
        if (i == BEACON_ID_THREAT) continue;
        if (rssi_table[i].last_seen_ms == 0) continue;
        if ((now - rssi_table[i].last_seen_ms) > RSSI_STALE_MS) continue;
        if (rssi_table[i].rssi_dbm > best_rssi) {
            best_rssi = rssi_table[i].rssi_dbm;
            best_id = i;
        }
    }
    if (best_id < 0) return -1;
    *out_id = (uint8_t)best_id;
    return 0;
}
\end{verbatim}
\end{enumerate}

\subsection{Step 4 --- DIO0 ISR}

The Threat's ISR differs from the cooperative-target ISR (LH1-F Step~3) in one important way: it does \emph{not} act on the packet contents, it only stamps and queues. All policy (forwarding, PAUSE-target selection, USART2 logging) happens in the main loop. This keeps the \ISR{} bounded to a few microseconds and prevents \mbox{RF} packet drops while the host \UART{} drains.

Add the following inside \texttt{/* USER CODE BEGIN 4 */}, after the ring buffer code from Step~2:

\begin{lstlisting}[language=C, caption={\texttt{HAL\_GPIO\_EXTI\_Callback} for the Threat. Stack-local struct, no printf, no main-loop policy. Reads RegFei[H/M/L] for the 24-bit signed frequency error and converts to int16 Hz.}]
/* SX1276 frequency-error registers; 24-bit signed in 0x1D..0x1F.
 * fei_hz = (raw * F_XOSC / 2^24) * (BW/500e3); for BW=125 kHz at
 * F_XOSC=32 MHz the fixed multiplier is ~1.907, so fei_hz_int16 =
 * raw * 2 (clipped) gives a usable two-byte field for downstream.
 */
static int16_t read_fei_hz(void) {
    uint8_t h = rfm_read_reg(0x1Du);
    uint8_t m = rfm_read_reg(0x1Eu);
    uint8_t l = rfm_read_reg(0x1Fu);
    int32_t raw = ((int32_t)(h & 0x0Fu) << 16)
                |  ((int32_t)m            <<  8)
                |   (int32_t)l;
    if (raw & 0x00080000) raw |= 0xFFF00000;     /* sign-extend 20-bit */
    int32_t hz = (raw * 32) / 1000;              /* coarse, BW-125 ok  */
    if (hz >  32767) hz =  32767;
    if (hz < -32768) hz = -32768;
    return (int16_t)hz;
}

void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    if (GPIO_Pin != RFM_DIO0_Pin) return;

    uint8_t irq = rfm_read_reg(0x12u);
    if ((irq & 0x40u) == 0u) {                   /* not RxDone */
        rfm_write_reg(0x12u, 0xFFu);
        return;
    }
    if (irq & 0x20u) {                           /* PayloadCrcError */
        rfm_write_reg(0x12u, 0xFFu);
        return;
    }

    intercepted_pkt_t pkt;
    uint8_t rx_addr = rfm_read_reg(0x10u);       /* RegFifoRxCurrentAddr */
    rfm_write_reg(0x0Du, rx_addr);               /* RegFifoAddrPtr */
    for (int i = 0; i < 32; i++) pkt.raw[i] = rfm_read_reg(0x00u);

    pkt.rx_timestamp_ms = ms_count;
    pkt.rssi_dbm        = (int8_t)((int16_t)rfm_read_reg(0x1Au) - 157);
    pkt.snr_q025        = (int8_t)rfm_read_reg(0x19u);
    pkt.freq_error_hz   = read_fei_hz();

    /* RSSI table update (Threat's job; not done in cooperative-target ISR) */
    uint8_t sender = pkt.raw[2];
    rssi_table[sender].rssi_dbm     = pkt.rssi_dbm;
    rssi_table[sender].last_seen_ms = pkt.rx_timestamp_ms;

    (void)ring_push(&pkt);                       /* dropped++ on overflow */
    rfm_write_reg(0x12u, 0xFFu);                 /* clear all IRQ flags */
}
\end{lstlisting}

\begin{tcolorbox}[warning={ISR-side SPI traffic is a real budget}]
The \ISR{} above issues $\sim$40 SPI byte transactions (32 FIFO bytes + RegIrqFlags + RegFifoRxCurrentAddr + RegFifoAddrPtr + RegPktRssiValue + RegPktSnrValue + 3 FEI registers + IRQ clear). At the LH1-C SPI prescaler of 32 (= 2.5\,MHz on a 80\,MHz APB2), each byte takes $\sim$3.2\,$\mu$s, so the \ISR{} runs $\sim$130\,$\mu$s. This is fine for a 2\,Hz ping-pong source but eats noticeable margin if you ever crank the source rate above $\sim$1\,kHz; if you do, switch to a SPI-DMA burst read of the 32-byte FIFO.
\end{tcolorbox}

\subsection{Step 5 --- PAUSE TX scheduler in the main loop}

The main loop has two responsibilities: drain the ring buffer (frame builder + UART forwarder) and poll the button (PAUSE TX scheduler). They share the radio in half-duplex, so the scheduler temporarily switches the radio out of \texttt{RXCONTINUOUS} for each PAUSE TX.

\begin{enumerate}[leftmargin=2em]
	\item Add the scheduler state near the top of the main loop:
\begin{verbatim}
uint32_t last_pause_tx_ms = 0;
uint16_t pause_seq = 0;
\end{verbatim}

	\item Inside the \texttt{while(1)} loop, add the scheduler logic after the ring-buffer drain:
\begin{verbatim}
/* --- Drain ring buffer and forward frames to FPGA --- */
intercepted_pkt_t pkt;
while (ring_pop(&pkt)) {
    uint8_t frame[43];
    build_forwarding_frame(frame, &pkt);  /* per Section "Forwarding Frame" */
    HAL_UART_Transmit_DMA(&huart3, frame, 43);
    /* wait for DMA complete or use a tx_busy flag */

    /* Optional debug print */
    uint8_t pkt_type = pkt.raw[6] & 0x03;
    printf("RX sender=%02X type=%u rssi=%d snr=%d\r\n",
           pkt.raw[2], pkt_type, (int)pkt.rssi_dbm,
           (int)(pkt.snr_q025 / 4));
}

/* --- PAUSE TX scheduler --- */
if (HAL_GPIO_ReadPin(B1_GPIO_Port, B1_Pin) == GPIO_PIN_RESET) {
    if ((ms_count - last_pause_tx_ms) >= 100) {
        uint8_t target_id;
        if (find_highest_rssi_id(&target_id) == 0) {
            /* Build PAUSE packet */
            uint8_t pkt_buf[32];
            build_packet(pkt_buf, BEACON_ID_THREAT, pause_seq,
                         PKT_TYPE_PAUSE, MOD_KIND_LORA);
            pkt_buf[13] = target_id;
            pkt_buf[14] = 5;  /* 5 * 100 ms = 500 ms hold-off */
            for (int i = 15; i < 30; i++) pkt_buf[i] = 0xAA;
            /* Recompute CRC-16 over bytes 0-29 */
            uint16_t crc = crc16_ccitt(pkt_buf, 30);
            pkt_buf[30] = (uint8_t)(crc >> 8);
            pkt_buf[31] = (uint8_t)(crc);

            /* Switch to STANDBY, TX the PAUSE, then back to RX */
            rfm_set_mode_standby();
            rfm_write_reg(0x0D, rfm_read_reg(0x0E));
            for (int i = 0; i < 32; i++) rfm_write_reg(0x00, pkt_buf[i]);
            rfm_write_reg(0x22, 32);
            rfm_write_reg(0x12, 0xFF);
            rfm_write_reg(0x01, 0x83);  /* TX */

            /* Wait for TxDone with timeout */
            uint32_t tx_deadline = ms_count + 200;
            while (ms_count < tx_deadline
                   && (rfm_read_reg(0x12) & 0x08) == 0) {
                /* spin */
            }
            rfm_write_reg(0x12, 0xFF);
            rfm_set_mode_rx();

            int8_t rssi = rssi_table[target_id].rssi_dbm;
            printf("PAUSE -> ID=%02X (RSSI=%d dBm, seq=%lu)\r\n",
                   target_id, (int)rssi, (unsigned long)pause_seq);

            last_pause_tx_ms = ms_count;
            pause_seq++;
        } else {
            /* No non-stale targets in the table */
            if ((ms_count - last_pause_tx_ms) >= 1000) {
                printf("no PAUSE target available\r\n");
                last_pause_tx_ms = ms_count;
            }
        }
    }
}
\end{verbatim}
\end{enumerate}

\subsection{Step 5b --- Periodic RSSI snapshot (live target diagnostics)}

The PAUSE TX path only logs when it actually emits a PAUSE, so if the button is not held --- or if the scheduler reports \texttt{"no PAUSE target available"} --- there is no visibility into \emph{why}: which Beacon~IDs the Threat is currently hearing, at what \RSSI{}, and which one would be selected if the button were pressed right now. Without that, common field scenarios (one target moves out of range, both targets briefly fall stale, one target dominates the other by 20\,dB) are invisible in the serial log.

Add a 1\,Hz snapshot to the main loop that dumps every Beacon~ID ever observed (with current \RSSI{} and elapsed time since last reception) and the result of an unconditional \texttt{find\_highest\_rssi\_id()} call. Declare a tracking timestamp alongside the Step~5 scheduler state:
\begin{verbatim}
uint32_t last_stats_ms = HAL_GetTick();   /* near last_pause_tx_ms */
\end{verbatim}
Then add the snapshot block at the bottom of the main \texttt{while(1)} loop, gated by a 1-second window:

\begin{lstlisting}[language=C, caption={Place at the bottom of the main \texttt{while(1)} loop, after the PAUSE TX scheduler block. Replaces any earlier 10-second \texttt{\#~stats} block.}]
/* --- Periodic stats + per-ID RSSI snapshot (every 1 s) --- */
if ((HAL_GetTick() - last_stats_ms) >= 1000u)
{
    uint32_t now = HAL_GetTick();
    last_stats_ms = now;

    printf("# stats: rx=%lu dropped=%lu\r\n",
           (unsigned long)rx_count,
           (unsigned long)dropped);

    /* Dump every Beacon ID we've ever seen, with current RSSI and the
     * elapsed time since the last reception. Stale entries (older than
     * RSSI_STALE_MS) are flagged so it's clear they would NOT be picked
     * by find_highest_rssi_id().                                       */
    for (int i = 1; i < 256; i++) {
        if (i == BEACON_ID_THREAT) continue;
        if (rssi_table[i].last_seen_ms == 0) continue;
        uint32_t age = now - rssi_table[i].last_seen_ms;
        const char *flag = (age > RSSI_STALE_MS) ? " [STALE]" : "";
        printf("# rssi[ID=0x%02X]=%d dBm age=%lu ms%s\r\n",
               (unsigned)i,
               (int)rssi_table[i].rssi_dbm,
               (unsigned long)age,
               flag);
    }

    /* What would the next PAUSE target be right now? */
    uint8_t cur_id;
    if (find_highest_rssi_id(&cur_id) == 0) {
        printf("# target: ID=0x%02X RSSI=%d dBm\r\n",
               (unsigned)cur_id,
               (int)rssi_table[cur_id].rssi_dbm);
    } else {
        printf("# target: none (no fresh observations)\r\n");
    }
}
\end{lstlisting}

Expected USART2 output during a healthy ping-pong run:
\begin{verbatim}
# stats: rx=42 dropped=0
# rssi[ID=0x01]=-87 dBm age=210 ms
# rssi[ID=0x02]=-65 dBm age=98 ms
# target: ID=0x02 RSSI=-65 dBm
\end{verbatim}

The three line types tell you, in order: how many packets the \ISR{} has captured and how many the main loop has dropped on overflow; one line per Beacon~ID that has ever been heard, with its most recent \RSSI{} and the age of that observation in milliseconds (\texttt{[STALE]} when \texttt{age > RSSI\_STALE\_MS}, meaning that \mbox{ID} is excluded from the highest-\RSSI{} lookup); and the unconditional result of \texttt{find\_highest\_rssi\_id()} --- i.e., which \mbox{ID} the Threat \emph{would} jam if you pressed the button right now. Watching the \texttt{\#~target:} line flip from \texttt{0x01} to \texttt{0x02} as you move Target~A away from the Threat is the canonical demonstration that re-acquisition is working; if the line stays at \texttt{0x01} or flips to \texttt{none}, the per-\mbox{ID} lines below it tell you why (e.g., Target~B never transmitted, so its entry was never populated; or both entries aged past \texttt{RSSI\_STALE\_MS}).

\begin{tcolorbox}[infobox={Tuning the staleness window}]
\texttt{RSSI\_STALE\_MS} defaults to 5000\,ms. At Target~A's 1\,Hz DATA cadence and Target~B's reactive ACK cadence, that tolerates 5 missed packets in a row before the entry drops out of selection. If the snapshot shows entries lingering at \texttt{[STALE]} longer than you want before re-acquisition, lower the threshold to 2500\,ms (2--3 missed packets). Conversely, if a marginal-link target keeps flickering in and out of selection, raise it. Document the chosen value alongside the ring-buffer-size rationale required by the deliverable.
\end{tcolorbox}

\subsection{Step 6 --- Frame builder, CRC-8, and UART forwarder}

The frame builder runs in main-loop context on each successful \texttt{ring\_pop()}. It writes into a single static buffer protected by a DMA-busy flag so the buffer is never overwritten while a previous \texttt{HAL\_UART\_Transmit\_DMA} is still in flight.

\begin{lstlisting}[language=C, caption={Add inside \texttt{/* USER CODE BEGIN 4 */}. CRC-8/ITU is a separate algorithm from LH1-F's CRC-16; do not share code.}]
/* CRC-8/ITU: poly=0x07, init=0x00, no reflect, xorout=0x00.
 * Verified against test vector: crc8_itu("123456789", 9) == 0xF4.
 */
static uint8_t crc8_itu(const uint8_t *buf, uint32_t len) {
    uint8_t crc = 0x00u;
    for (uint32_t i = 0; i < len; i++) {
        crc ^= buf[i];
        for (int b = 0; b < 8; b++) {
            crc = (crc & 0x80u) ? (uint8_t)((crc << 1) ^ 0x07u)
                                : (uint8_t)(crc << 1);
        }
    }
    return crc;
}

/* Builds the 43-byte forwarding frame per LH1-G Table on page 2.
 * The embedded radio-packet CRC-16 (already in pkt->raw[30..31]) is
 * copied through unchanged at frame[40..41]; the frame CRC-8 at
 * frame[42] is computed over frame[1..41] (length, metadata, packet).
 */
static void build_forwarding_frame(uint8_t out[43],
                                   const intercepted_pkt_t *pkt)
{
    out[0] = 0x7Eu;                              /* start delimiter */
    out[1] = 0x29u;                              /* length = 41 */

    /* Timestamp, little-endian */
    out[2] = (uint8_t)(pkt->rx_timestamp_ms      );
    out[3] = (uint8_t)(pkt->rx_timestamp_ms >>  8);
    out[4] = (uint8_t)(pkt->rx_timestamp_ms >> 16);
    out[5] = (uint8_t)(pkt->rx_timestamp_ms >> 24);

    out[6] = (uint8_t)pkt->rssi_dbm;             /* int8 two's complement */
    out[7] = (uint8_t)pkt->snr_q025;

    out[8] = (uint8_t)(pkt->freq_error_hz      );
    out[9] = (uint8_t)(pkt->freq_error_hz >>  8);

    /* Embedded 32-byte radio packet (DATA, ACK, or PAUSE) verbatim */
    for (int i = 0; i < 32; i++) out[10 + i] = pkt->raw[i];

    out[42] = crc8_itu(&out[1], 41);             /* over bytes 1..41 */
}
\end{lstlisting}

The \mbox{DMA} busy flag pattern protects the 43-byte transmit buffer:

\begin{lstlisting}[language=C, caption={Add to \texttt{/* USER CODE BEGIN PV */} (storage) and \texttt{/* USER CODE BEGIN 4 */} (callback).}]
/* --- USER CODE BEGIN PV --- */
static uint8_t  tx_frame[43];
volatile uint8_t tx_busy = 0;
/* --- USER CODE END PV --- */

/* --- USER CODE BEGIN 4 --- */
void HAL_UART_TxCpltCallback(UART_HandleTypeDef *huart) {
    if (huart->Instance == USART3) tx_busy = 0;
}
/* --- USER CODE END 4 --- */
\end{lstlisting}

Then the main-loop drain (already shown in Step~5) becomes:

\begin{lstlisting}[language=C, caption={Replace the placeholder \texttt{HAL\_UART\_Transmit\_DMA} call inside the Step~5 ring-drain block with this gated version.}]
intercepted_pkt_t pkt;
while (ring_pop(&pkt)) {
    /* Wait for any prior DMA to complete (with a short timeout to avoid
     * deadlock if USART3 wedges).  At 115200 baud, 43 bytes = ~3.7 ms. */
    uint32_t deadline = ms_count + 10u;
    while (tx_busy && ms_count < deadline) { /* spin */ }
    if (tx_busy) { dropped++; continue; }     /* give up; record drop */

    build_forwarding_frame(tx_frame, &pkt);
    tx_busy = 1;
    HAL_UART_Transmit_DMA(&huart3, tx_frame, 43);

    uint8_t pkt_type = pkt.raw[6] & 0x03u;
    printf("RX sender=%02X type=%u rssi=%d snr=%d\r\n",
           pkt.raw[2], pkt_type, (int)pkt.rssi_dbm,
           (int)(pkt.snr_q025 / 4));
}
\end{lstlisting}

\begin{tcolorbox}[warning={Do not reuse LH1-F's \texttt{crc16\_ccitt} for the frame CRC}]
The \emph{embedded radio-packet} CRC at bytes 30--31 of the radio packet is CRC-16/CCITT-FALSE, computed by \texttt{crc16\_ccitt} inside \texttt{build\_packet} (LH1-F). The \emph{forwarding frame} CRC at byte 42 of the 43-byte UART frame is CRC-8/ITU, computed by \texttt{crc8\_itu} above. They are different polynomials, different widths, and protect different things. The Module~2 parser checks both independently --- mixing them up will cause silent CRC failures on the FPGA.
\end{tcolorbox}

\subsection{Step 7 --- Validation without the \FPGA{} (optional)}

This step provides an early sanity check that the Threat is producing well-formed 43-byte frames with both CRCs validating cleanly, \emph{before} you bring the \FPGA{} into the loop. It requires a USB-to-UART adapter; if you do not have one, skip directly to Lab Handout~1-H, where the same forwarding stream is validated end-to-end via the Nexys~A7 \mbox{PMOD} and the on-board seven-segment display.

\begin{tcolorbox}[infobox={Wiring for Step 7 (USB-UART Adapter)}]
\begin{itemize}[leftmargin=1.2em]
	\item Threat \texttt{USART3 TX} = \texttt{PC4} $\to$ adapter \texttt{RX}
	\item Threat \texttt{GND} $\to$ adapter \texttt{GND}
	\item Adapter \texttt{TX} is unused (the Threat does not receive on \texttt{USART3}).
	\item Logic level: 3.3\,V on both ends. Do \emph{not} use a 5\,V-only TTL adapter without a level shifter.
	\item Baud: \texttt{115200}, 8N1, no flow control.
	\item The adapter enumerates as a separate \mbox{COM}/tty device --- point the parser at \emph{that} port, not the Nucleo's ST-Link port.
\end{itemize}
\end{tcolorbox}

Save this as \texttt{tools/lh1g\_parser.py} in the project root:

\begin{lstlisting}[language=Python, caption={\texttt{tools/lh1g\_parser.py} --- run with \texttt{python lh1g\_parser.py COM7 115200 LH1G\_capture.csv}.}]
"""LH1-G forwarding-frame parser.

Reads 43-byte frames from a serial port, validates the frame CRC-8 (byte 42)
and the embedded radio-packet CRC-16 (packet bytes 30..31), decodes the
packed Modulation Code byte, and writes one CSV row per valid frame.
"""
import csv
import struct
import sys
from pathlib import Path

import serial


def crc16_ccitt(data: bytes) -> int:
    """CRC-16/CCITT-FALSE: poly=0x1021, init=0xFFFF, no reflect."""
    crc = 0xFFFF
    for b in data:
        crc ^= b << 8
        for _ in range(8):
            crc = ((crc << 1) ^ 0x1021) & 0xFFFF if crc & 0x8000 \
                else (crc << 1) & 0xFFFF
    return crc


def crc8_itu(data: bytes) -> int:
    """CRC-8/ITU: poly=0x07, init=0x00, no reflect."""
    crc = 0
    for b in data:
        crc ^= b
        for _ in range(8):
            crc = ((crc << 1) ^ 0x07) & 0xFF if crc & 0x80 \
                else (crc << 1) & 0xFF
    return crc


PKT_TYPE_NAMES = {0: "DATA", 1: "ACK", 2: "PAUSE", 3: "RESERVED"}
MOD_KIND_NAMES = {1: "LoRa", 2: "FSK", 3: "OOK"}


def parse_frame(buf: bytes) -> dict | None:
    """Validate one 43-byte frame; return decoded fields or None on error."""
    if len(buf) != 43 or buf[0] != 0x7E or buf[1] != 0x29:
        return {"error": "framing"}
    if crc8_itu(buf[1:42]) != buf[42]:
        return {"error": "frame_crc8"}

    rx_ts_ms = struct.unpack("<I", buf[2:6])[0]
    rssi     = struct.unpack("b",  buf[6:7])[0]
    snr_q025 = struct.unpack("b",  buf[7:8])[0]
    freq_err = struct.unpack("<h", buf[8:10])[0]
    radio    = buf[10:42]                      # 32-byte radio packet

    if crc16_ccitt(radio[:30]) != ((radio[30] << 8) | radio[31]):
        return {"error": "radio_crc16"}

    mod_byte  = radio[6]
    mod_kind  = (mod_byte >> 2) & 0x3F
    pkt_type  =  mod_byte       & 0x03
    sender    = radio[2]
    seq       = radio[3] | (radio[4] << 8)

    return {
        "t_ms":     rx_ts_ms,
        "sender":   f"0x{sender:02X}",
        "pkt_type": PKT_TYPE_NAMES.get(pkt_type, "?"),
        "mod_kind": MOD_KIND_NAMES.get(mod_kind, f"?({mod_kind})"),
        "seq":      seq,
        "rssi":     rssi,
        "snr_db":   snr_q025 / 4.0,
        "freq_err": freq_err,
    }


def main(port: str, baud: int, out_csv: str) -> int:
    ser = serial.Serial(port, baud, timeout=0.2)
    fields = ["t_ms", "sender", "pkt_type", "mod_kind",
              "seq", "rssi", "snr_db", "freq_err"]
    n_ok = n_bad = 0

    with Path(out_csv).open("w", newline="") as f:
        writer = csv.DictWriter(f, fieldnames=fields)
        writer.writeheader()
        buf = bytearray()
        try:
            while True:
                buf += ser.read(64)
                while True:
                    # find next start delimiter
                    i = buf.find(b"\x7E")
                    if i < 0:
                        buf.clear(); break
                    if i > 0:
                        del buf[:i]
                    if len(buf) < 43:
                        break
                    frame = bytes(buf[:43])
                    del buf[:43]
                    decoded = parse_frame(frame)
                    if decoded and "error" not in decoded:
                        writer.writerow(decoded); f.flush()
                        n_ok += 1
                        print(decoded)
                    else:
                        n_bad += 1
                        print(f"DROP {decoded}", file=sys.stderr)
        except KeyboardInterrupt:
            pass
    print(f"Parsed {n_ok} ok, {n_bad} dropped")
    return 0


if __name__ == "__main__":
    if len(sys.argv) != 4:
        print("usage: lh1g_parser.py <port> <baud> <out_csv>")
        sys.exit(2)
    sys.exit(main(sys.argv[1], int(sys.argv[2]), sys.argv[3]))
\end{lstlisting}

\section{Verification Checklist}
\begin{itemize}[leftmargin=2em]
	\item[$\square$] \texttt{RxDone} interrupt fires on every received packet (\ISR{} entry count matches expected receptions).
	\item[$\square$] Ring buffer overflows only if you deliberately stall the main loop.
	\item[$\square$] 43-byte frames appear on \texttt{USART3 TX} for every received DATA, ACK, and PAUSE.
	\item[$\square$] Host Python parser validates start delimiter, byte[1]=\texttt{0x29}, frame \CRC{}-8, and embedded radio-packet \CRC{}-16 on 100\% of frames.
	\item[$\square$] Ping-pong run: parser sees interleaved \mbox{ID}\,=\,\texttt{0x01} (DATA, type=0) and \mbox{ID}\,=\,\texttt{0x02} (ACK, type=1) frames with matching sequence numbers.
	\item[$\square$] RSSI table populated with both Targets' entries within 1\,s of starting the ping-pong source.
	\item[$\square$] 1\,Hz snapshot (Step~5b) prints \texttt{\#~stats}, one \texttt{\#~rssi[ID=0xNN]} line per ever-seen Beacon~\mbox{ID} with current \RSSI{} and \texttt{age=}, and a \texttt{\#~target:} line; entries past \texttt{RSSI\_STALE\_MS} are flagged \texttt{[STALE]}.
	\item[$\square$] Re-acquisition test: \texttt{\#~target:} line flips from \texttt{0x01} to \texttt{0x02} (or back) within \texttt{RSSI\_STALE\_MS} when one Target is removed, and the very next button-held PAUSE addresses the new target.
	\item[$\square$] Button-held PAUSE TX: USART2 log shows \texttt{PAUSE -> ID=...} at $\sim$10\,Hz with the higher-\RSSI{} Target ID.
	\item[$\square$] Co-validated with LH1-F: targeted Target prints \texttt{"I'm Jammed"} on its USART2 within 100\,ms of button press.
	\item[$\square$] Dropped-packet counter remains zero during a 5-minute run.
\end{itemize}

\begin{tcolorbox}[warning={Concurrency Hazards}]
	\begin{itemize}[leftmargin=1.2em]
		\item Access \texttt{ring\_head}/\texttt{ring\_tail} as \texttt{volatile}. Never read the same index twice in the consumer without rechecking.
		\item Do not call \HAL{} \texttt{printf} from inside the \ISR{} --- it blocks on \mbox{UART} transmit and will drop \RF{} packets.
		\item Do not overwrite the \mbox{DMA} source buffer while a prior transmission is still in flight. Use a simple busy flag or a second buffer.
		\item The PAUSE TX scheduler temporarily switches the radio out of \texttt{RXCONTINUOUS}; any DATA/ACK arriving during that $\sim$50\,ms TX window is missed. This is the documented half-duplex cost and is the right behavior --- it is the reason the framework records the PAUSE-active period as a measurement consideration in Module~5 rather than treating the missed receptions as a bug.
	\end{itemize}
\end{tcolorbox}
