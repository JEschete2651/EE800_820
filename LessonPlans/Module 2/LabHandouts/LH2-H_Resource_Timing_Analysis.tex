\labheader{2-H}{Resource Utilization, Timing Analysis, and Bitstream Sign-Off}

\section{Objective}
Close Module~2 by running the full \texttt{ingest\_top} design through Vivado implementation, analyzing the resource and timing reports, fixing any remaining negative slack, and producing a signed-off bitstream ready for Module~3 system integration.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handout~2-G (end-to-end simulation passing all three scenarios).
\end{itemize}

\section{Procedure}

\subsection{Step 1 --- Run implementation}
\begin{enumerate}[leftmargin=2em]
	\item Flow Navigator $\to$ \textbf{Run Implementation}. Wait for completion.
	\item If synthesis or implementation errors appear, fix them before proceeding. Warnings about unused signals are acceptable; warnings about inferred latches are not.
	\item Open the \textbf{Utilization} report. Record \mbox{LUT}, \mbox{FF}, \BRAM{}, and \DSP{} usage.
\end{enumerate}

\begin{tcolorbox}[infobox={Expected utilization envelope}]
	For the reference implementation on the Artix-7 100T, expect roughly: \mbox{LUT} 1--3\,\%, \mbox{FF} 0.5--2\,\%, \BRAM{} 2 tiles, \DSP{} 0. Numbers substantially higher than this usually point at an accidental block-\mbox{RAM} inference or a combinational feedback loop.
\end{tcolorbox}

\subsection{Step 2 --- Timing report}
\begin{enumerate}[leftmargin=2em]
	\item Open \textbf{Report Timing Summary}. Worst Negative Slack (WNS) must be $\geq 0$ for both setup and hold.
	\item If setup slack is negative, pull up the top 10 failing paths. Typical culprits: the \CRC{} XOR tree (see 2-E), a long combinational compare in the parser \FSM{}, or an unregistered output into \texttt{led}.
	\item Register any failing output. Re-run implementation and confirm positive slack.
\end{enumerate}

\labfigure{LH2H_timing_summary.png}{0.85\textwidth}{Vivado Timing Summary report with all four categories (WNS/TNS, WHS/THS, WPWS/TPWS) showing positive slack.}{fig:lh2h-timing}

\subsection{Step 3 --- Clock domain crossings}
Verify that Vivado's \textbf{Report CDC} shows only the expected crossing: \texttt{uart\_rxd} into \texttt{sysclk} via \texttt{sync\_2ff}. If any unexpected crossings appear, trace them before proceeding.

\subsection{Step 4 --- Generate the bitstream}
\begin{enumerate}[leftmargin=2em]
	\item Click \textbf{Generate Bitstream}. The output is \texttt{ingest\_top.bit}.
	\item Open the \textbf{Power} report. Dynamic power should be under 200\,mW for this design.
\end{enumerate}

\subsection{Step 5 --- Hardware sanity check}
\begin{enumerate}[leftmargin=2em]
	\item Program the Nexys A7 from Hardware Manager.
	\item Connect the STM32 interceptor \UART{} \texttt{TX} to the Pmod JA pin declared in \texttt{ingest\_top.xdc}.
	\item Power on the cooperative-target pair (Target~A and Target~B) plus the interceptor (Threat). The \texttt{led(0)} \FIFO{}-empty \mbox{LED} should blink as bytes arrive; \texttt{led(1)} (\FIFO{}-full) should stay dark.
	\item Add an \ILA{} probe on \texttt{frame\_valid} and \texttt{frame\_reject}. Confirm that \texttt{frame\_valid} pulses at roughly the Module~1 ping-pong cadence (DATA $\to$ ACK pairs) and \texttt{frame\_reject} is rare.
\end{enumerate}

\subsection{Step 6 --- Export reports to machine-readable format}
The Module~5 system specification table (and the Module~2 resource row in the final report) requires exact \mbox{LUT}/\mbox{FF}/\BRAM{}/WNS numbers. Export them from Vivado's Tcl console now so they are available without re-opening the GUI later.

\begin{lstlisting}[language=Tcl, caption={Run in the Vivado Tcl Console after implementation completes.}]
# Open the implemented design if not already open
open_run impl_1

# Export utilization to CSV
report_utilization -hierarchical -file impl/util.rpt

# Export timing summary
report_timing_summary -file impl/timing.rpt -warn_on_violation

# Export power
report_power -file impl/power.rpt
\end{lstlisting}

Then run the following Python snippet once to extract the numbers into a structured \texttt{impl\_summary.json} that the Module~5 spec-table script can read directly:

\begin{lstlisting}[language=Python, caption={\texttt{parse\_vivado\_reports.py} --- run from the \texttt{impl/} directory.}]
import json, re
from pathlib import Path

def parse_util(rpt: str) -> dict:
    patterns = {
        "lut":  r"Slice LUTs\s*\|\s*(\d+)",
        "ff":   r"Slice Registers\s*\|\s*(\d+)",
        "bram": r"Block RAM Tile\s*\|\s*(\d+)",
        "dsp":  r"DSPs\s*\|\s*(\d+)",
    }
    result = {}
    for key, pat in patterns.items():
        m = re.search(pat, rpt)
        result[key] = int(m.group(1)) if m else None
    return result

def parse_timing(rpt: str) -> dict:
    m = re.search(r"WNS\(ns\)\s+TNS\(ns\).*?\n\s*([-\d.]+)\s+([-\d.]+)", rpt, re.S)
    return {"wns_ns": float(m.group(1)) if m else None,
            "tns_ns": float(m.group(2)) if m else None}

util = parse_util(Path("util.rpt").read_text(errors="replace"))
timing = parse_timing(Path("timing.rpt").read_text(errors="replace"))

summary = {"module": "Module2_ingest_top", **util, **timing}
Path("impl_summary.json").write_text(json.dumps(summary, indent=2))
print(json.dumps(summary, indent=2))
\end{lstlisting}

Example output:
\begin{lstlisting}
{
  "module": "Module2_ingest_top",
  "lut": 312,
  "ff": 198,
  "bram": 2,
  "dsp": 0,
  "wns_ns": 2.14,
  "tns_ns": 0.0
}
\end{lstlisting}

Commit \texttt{impl\_summary.json} alongside the bitstream. This file is the authoritative source for Table~2 in the final report.

\subsection{Step 7 --- Archive the sign-off artifacts}
Collect into a single directory:
\begin{itemize}[leftmargin=2em]
	\item \texttt{ingest\_top.bit}.
	\item \texttt{impl\_summary.json} (from Step~6).
	\item The raw \texttt{util.rpt}, \texttt{timing.rpt}, and \texttt{power.rpt} files.
	\item The Module~2-G regression results (diff files from all three scenarios).
	\item A one-page sign-off summary confirming \mbox{LUT}/\mbox{FF}/\BRAM{} within the expected envelope, WNS $\geq$ 0, and dynamic power $<$ 200\,mW.
\end{itemize}

\section{Verification Checklist}
\begin{itemize}[leftmargin=2em]
	\item[$\square$] Implementation completes with zero errors and no inferred-latch warnings.
	\item[$\square$] WNS and WHS both $\geq 0$.
	\item[$\square$] Report CDC shows only the \texttt{uart\_rxd} $\to$ \texttt{sysclk} crossing.
	\item[$\square$] Dynamic power $<$ 200\,mW.
	\item[$\square$] Hardware \ILA{} capture confirms \texttt{frame\_valid} activity on a live cooperative-target ping-pong stream (with PAUSE bursts visible when the Threat user button is held).
\end{itemize}

\begin{tcolorbox}[warning={Common Pitfalls}]
	\begin{itemize}[leftmargin=1.2em]
		\item \textbf{Negative hold slack}: almost always a missing register between two ultra-short paths. Add a single flip-flop rather than trying to slow the clock.
		\item \textbf{Inferred latch warning}: an incomplete \texttt{if}/\texttt{case} in a combinational process. Provide default assignments at the top of every combinational block.
		\item \textbf{Unexpected \BRAM{} inference}: the parser's 32-byte \texttt{payload} register array accidentally sized above Vivado's distributed-\mbox{RAM} threshold. Force distributed \mbox{RAM} with the \texttt{(*ram\_style = "distributed"*)} attribute.
	\end{itemize}
\end{tcolorbox}

	\end{enumerate}
