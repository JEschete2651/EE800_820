\labheader{2-G}{End-to-End Pipeline Simulation with Recorded Beacon Data}

\section{Objective}
Stand up a single testbench that drives the complete Module~2 pipeline --- \texttt{uart\_rx} $\to$ \texttt{byte\_fifo} $\to$ \texttt{pkt\_parser} $\to$ \texttt{crc} engines $\to$ \texttt{feature\_extract} $\to$ \texttt{feature\_bram} --- using a recorded Module~1 capture, and compare every exported feature vector against a Python reference.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handouts~2-A through 2-F (full pipeline integrated under \texttt{ingest\_top}).
	\item Recorded multi-beacon Module~1 capture (\texttt{.bin}) with at least 1000 frames.
\end{itemize}

\section{Procedure}

\subsection{Step 1 --- Scenario selection}
Pick three test captures from Module~1:
\begin{enumerate}[leftmargin=2em]
	\item \textbf{Clean single-beacon}: one emitter, no dropped frames. Validates the happy path.
	\item \textbf{Multi-beacon mix}: three or more emitters transmitting in overlapping time windows. Validates beacon-ID dispatch and \BRAM{} write arbitration.
	\item \textbf{Corrupted stream}: intentionally introduce bit flips in a small fraction of frames (pre-process the \texttt{.bin} with a short Python script). Validates the reject path.
\end{enumerate}

\subsection{Step 2 --- Testbench harness}
Create \texttt{sim/tb\_ingest\_top.vhd}:
\begin{enumerate}[leftmargin=2em]
	\item Generate a 100\,MHz \texttt{sysclk} and release \texttt{rstn} after 100\,ns.
	\item Open a \texttt{.bin} file with \texttt{std.textio}. Transmit every byte over \texttt{uart\_rxd} at exactly 115200\,baud using a bit-banging process.
	\item Snoop the parser signals \texttt{frame\_valid}/\texttt{frame\_reject}. Log each event with the simulation timestamp.
	\item After end-of-stream, walk the \texttt{feature\_bram} read port over all 256 rows and dump the contents to a text file.
\end{enumerate}

\subsection{Step 3 --- Python reference}
Write \texttt{sim/ref\_pipeline.py} that ingests the same \texttt{.bin}, applies the identical frame parser and feature rules, and produces an expected-feature dump in the same text format.

\subsection{Step 4 --- Diff and pass criteria}
Diff the simulation dump against the Python reference. Pass criteria:
\begin{itemize}[leftmargin=2em]
	\item \textbf{Clean single-beacon}: byte-for-byte match on all non-reserved fields.
	\item \textbf{Multi-beacon mix}: byte-for-byte match on all occupied rows.
	\item \textbf{Corrupted stream}: the number of \texttt{frame\_reject} events equals the number of frames with injected bit flips, and no corrupted frame produced a feature update.
\end{itemize}

\subsection{Step 5 --- Runtime and waveform capture}
A 1000-frame capture at 115200\,baud is roughly 4\,s of wall-clock time in simulation on a modern host. Record the simulation runtime and store the waveform for the first 10 frames and for any rejected frame.

\begin{tcolorbox}[warning={Simulation runtime trap}]
	Full-waveform logging across a 4\,s simulation is slow and disk-hungry. Use Vivado's \texttt{log\_wave -r /tb\_ingest\_top/uut/*} only for targeted windows, or drop to a headless \texttt{xsim} run with waveform logging disabled for the main regression.
\end{tcolorbox}

\section{Verification Checklist}
\begin{itemize}[leftmargin=2em]
	\item[$\square$] Clean-capture test: pipeline dump matches Python reference with zero diffs.
	\item[$\square$] Multi-beacon test: occupied rows match; unoccupied rows remain all-zero.
	\item[$\square$] Corrupted-stream test: reject count matches injected-bit-flip count.
	\item[$\square$] Waveform captures saved for the first 10 frames and for one rejected frame.
\end{itemize}

\begin{tcolorbox}[deliverable={Lab Handout 2-G Deliverable}]
	\begin{enumerate}[leftmargin=1.2em]
		\item \texttt{tb\_ingest\_top.vhd} and \texttt{ref\_pipeline.py}.
		\item Diff result files for all three scenarios.
		\item Waveform screenshots of a valid frame sequence and a rejected frame.
	\end{enumerate}
\end{tcolorbox}
