\labheader{2-F}{Feature Extraction Datapath and Feature \BRAM{}}

\section{Objective}
Transform each validated frame into a fixed 32-byte feature vector and store it in a dual-port \BRAM{} indexed by Beacon~ID. By the end of this handout the pipeline will accumulate features per station across multiple frames and expose a read port for the Module~3 host interface. The feature vector includes the bit-field-extracted \texttt{pkt\_type} column (low 2 bits of the radio packet's \texttt{Modulation Code} byte: \texttt{0x00}=DATA, \texttt{0x01}=ACK, \texttt{0x02}=PAUSE) and, when applicable, the PAUSE-only fields \texttt{target\_id\_if\_pause} and \texttt{pause\_duration\_units} extracted from the radio payload.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handout~2-E (parser emits \texttt{frame\_valid} only for double-\CRC{}-clean frames).
\end{itemize}

\section{Feature Vector Layout}
The feature vector is 32~bytes, aligned to one \BRAM{} row. The layout matches \texttt{tab:feature\_vector} in the Module~2 lesson plan and \texttt{tab:featvec} in the final report --- do not reorder fields, since the Module~4 schema manifest validates this layout byte-for-byte at deployment time.
\begin{center}
\begin{tabular}{c l l l}
\toprule
\textbf{Offset} & \textbf{Field} & \textbf{Size} & \textbf{Source} \\
\midrule
0      & Beacon ID                       & 1\,B & radio packet byte 2 \\
1      & Modulation Kind                 & 1\,B & (radio packet byte 6 $\gg$ 2) \& \texttt{0x3F} (low 2 bits cleared) \\
2      & Spreading Factor                & 1\,B & radio packet byte 7 \\
3      & \texttt{pkt\_type}              & 1\,B & radio packet byte 6 \& \texttt{0x03} (\texttt{0x00}=DATA, \texttt{0x01}=ACK, \texttt{0x02}=PAUSE) \\
4--5   & \RSSI{} (latest)                & 2\,B & metadata RSSI sign-extended \\
6--7   & \SNR{} (latest)                 & 2\,B & metadata SNR sign-extended \\
8--11  & Frame count (for this ID)       & 4\,B & internal counter \\
12--15 & Last arrival timestamp          & 4\,B & metadata RX timestamp (ms) \\
16--19 & Inter-arrival time (ticks)      & 4\,B & computed \\
20--23 & \RSSI{} delta                   & 4\,B & computed \\
24--27 & Priority score                  & 4\,B & computed \\
28     & \texttt{target\_id\_if\_pause}  & 1\,B & radio payload byte 0 if \texttt{pkt\_type} = PAUSE; \texttt{0x00} otherwise \\
29     & \texttt{pause\_duration\_units} & 1\,B & radio payload byte 1 if \texttt{pkt\_type} = PAUSE; \texttt{0x00} otherwise \\
30--31 & Reserved / padding              & 2\,B & --- \\
\bottomrule
\end{tabular}
\end{center}

\section{Procedure}

\subsection{Step 1 --- Allocate the \BRAM{}}
Create \texttt{rtl/feature\_bram.vhd} instantiating an inferred dual-port \BRAM{}: 256 rows $\times$ 256 bits (32~bytes), write port driven by the feature extractor, read port driven by the Module~3 host interface (left stubbed in this lab).

\begin{tcolorbox}[infobox={Why 256 rows?}]
	Beacon~ID is 1 byte, so 256 rows accommodates every possible sender with no hashing. The bench scenario uses three IDs (Target~A=\texttt{0x01}, Target~B=\texttt{0x02}, Threat=\texttt{0xFF}), but the addressable space scales without code changes. At 256$\times$32\,B $=$ 8\,KiB, the array fits in exactly two 36-Kb \BRAM{} tiles (out of 135 available), leaving ample headroom for future features.
\end{tcolorbox}

\subsection{Step 2 --- Feature extractor}
Create \texttt{rtl/feature\_extract.vhd}. On every \texttt{frame\_valid} pulse:
\begin{enumerate}[leftmargin=2em]
	\item Latch the incoming \texttt{payload} (32-byte radio packet) and \texttt{metadata} (RX timestamp, RSSI, SNR, freq.\ error from the forwarding frame).
	\item Read the current feature row for the incoming Beacon~ID from the \BRAM{}.
	\item Extract bit-field-packed values from \texttt{payload(6)} (the radio packet's \texttt{Modulation Code} byte): \texttt{pkt\_type = payload(6) and "00000011"}, \texttt{mod\_kind = "00" \& payload(6)(7 downto 2) shifted to a clean byte value} so the result equals the canonical \texttt{0x01}/\texttt{0x02}/\texttt{0x03} for LoRa/FSK/OOK with the low 2 bits cleared.
	\item Compute inter-arrival time: current timestamp minus stored last timestamp.
	\item Compute \RSSI{} delta: current \RSSI{} minus stored \RSSI{}.
	\item Compute priority score: a fixed linear combination of \RSSI{}, \SNR{}, and frame count (weights defined in Module~2 Section~4).
	\item Conditionally extract the PAUSE-only fields: if \texttt{pkt\_type = "10"} (PAUSE), copy \texttt{payload(13)} into feature offset~28 (\texttt{target\_id\_if\_pause}) and \texttt{payload(14)} into feature offset~29 (\texttt{pause\_duration\_units}); otherwise zero both fields.
	\item Assemble the 32-byte vector per the layout above and write it back to the same row.
\end{enumerate}
The read-modify-write cycle takes a fixed three clock cycles; document that latency at the module port.

\labfigure{LH2F_feature_datapath.png}{0.85\textwidth}{Feature extraction datapath. The \BRAM{} read port feeds the compute block; the write port commits the updated vector on the following cycle.}{fig:lh2f-datapath}

\subsection{Step 3 --- Backpressure}
The parser must not issue a new \texttt{frame\_valid} while the feature extractor is mid-update. Add a \texttt{feat\_busy} signal from the extractor and gate the parser's \texttt{HUNT} $\to$ \texttt{LEN} transition on \texttt{feat\_busy = 0}.

\subsection{Step 4 --- Feature extractor testbench}
Create \texttt{sim/tb\_feature\_extract.vhd}. Inject five frames covering all three packet types: three DATA from Beacon~ID \texttt{0x01}, two ACK from Beacon~ID \texttt{0x02}, and one PAUSE from Beacon~ID \texttt{0xFF} addressed to Target~A (\texttt{target\_id} = \texttt{0x01}, \texttt{duration\_units} = \texttt{0x05}). After the stimulus, read rows \texttt{0x01}, \texttt{0x02}, and \texttt{0xFF} through the \BRAM{} read port and verify:
\begin{enumerate}[leftmargin=2em]
	\item Frame counts are 3, 2, and 1 respectively.
	\item Row \texttt{0x01}'s \texttt{pkt\_type} field (offset~3) is \texttt{0x00} (DATA); row \texttt{0x02}'s is \texttt{0x01} (ACK); row \texttt{0xFF}'s is \texttt{0x02} (PAUSE).
	\item Modulation Kind field (offset~1) equals \texttt{0x01} (LoRa) on all three rows --- the low 2 bits of the source byte are correctly masked off.
	\item Row \texttt{0xFF}'s \texttt{target\_id\_if\_pause} (offset~28) equals \texttt{0x01} and \texttt{pause\_duration\_units} (offset~29) equals \texttt{0x05}; rows \texttt{0x01} and \texttt{0x02} have both fields at \texttt{0x00}.
	\item Inter-arrival time equals the difference between the second-to-last and last timestamps you injected.
	\item Priority scores match a Python reference implementation.
\end{enumerate}

\section{Verification Checklist}
\begin{itemize}[leftmargin=2em]
	\item[$\square$] \BRAM{} inferred as two 36-Kb tiles in post-synthesis utilization.
	\item[$\square$] Feature extractor produces the three computed fields matching the Python reference.
	\item[$\square$] \texttt{pkt\_type} field (offset~3) correctly encodes DATA / ACK / PAUSE for the three test stations.
	\item[$\square$] Modulation Kind field (offset~1) has the low 2 bits cleared.
	\item[$\square$] PAUSE-only fields (offsets~28--29) populate only on PAUSE rows; zero otherwise.
	\item[$\square$] Backpressure (\texttt{feat\_busy}) prevents parser overrun during a burst.
	\item[$\square$] Frame count, \RSSI{}, and \SNR{} fields update correctly across multiple frames per station.
\end{itemize}
