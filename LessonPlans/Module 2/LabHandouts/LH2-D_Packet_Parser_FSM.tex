\labheader{2-D}{Packet Parser Finite State Machine}

\section{Objective}
Implement the \VHDL{} parser that drains \texttt{byte\_fifo}, locks onto the Module~1 receiver forwarding frame (start delimiter \texttt{0x7E}, 1-byte length, 8-byte metadata, 32-byte embedded beacon payload, 1-byte \CRC{}-8), and produces registered outputs for each field. By the end of this handout a valid frame will emit a \texttt{frame\_valid} pulse with the parsed fields stable on the same cycle.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handout~2-C (Week~1 pipeline integration verified).
	\item Module~1 receiver frame format reference (43 bytes total).
\end{itemize}

\section{Background}
The receiver forwarding frame is defined by the Module~1 lesson plan (Table~6) and is the single canonical spec across Modules~1--5. Reproduced verbatim below:
\begin{center}
\begin{tabular}{c l c l}
\toprule
\textbf{Offset} & \textbf{Field} & \textbf{Size} & \textbf{Notes} \\
\midrule
0     & Start delimiter \texttt{0x7E}                         & 1\,B & Frame sync \\
1     & Length byte                                           & 1\,B & Must equal \texttt{0x29} (41 decimal) \\
2--5  & RX timestamp (uint32, \mbox{LE}, ms)                  & 4\,B & Metadata \\
6     & \RSSI{} (int8, dBm)                                   & 1\,B & Metadata \\
7     & \SNR{} (int8, 0.25\,dB)                               & 1\,B & Metadata \\
8--9  & Frequency error (int16, \mbox{LE}, Hz)                & 2\,B & Metadata \\
10--41 & Beacon payload (32-byte beacon packet)               & 32\,B & Includes embedded \CRC{}-16 at beacon bytes 30--31 = frame 40--41 \\
42    & Frame \CRC{}-8/ITU (poly \texttt{0x07}) over bytes 1--41 & 1\,B & Transport integrity \\
\bottomrule
\end{tabular}
\end{center}

\begin{tcolorbox}[infobox={Length-Field Rule}]
The length byte equals the count of bytes \emph{excluding the start delimiter and the trailing \CRC{}}. For a 43-byte frame, byte[1] must equal \texttt{0x29} (41 decimal). Reject any frame where byte[1] $\neq$ \texttt{0x29}.
\end{tcolorbox}

Byte 42 is the frame \CRC{}-8/ITU computed over bytes 1--41 (not 0--41 --- the start delimiter is excluded). The beacon payload also carries its own \CRC{}-16/CCITT-FALSE internally at beacon bytes 30--31 (frame offsets 40--41), checked in Lab Handout~2-E.

\begin{tcolorbox}[infobox={Pre-Check: Known-Good Reference Frame}]
Before wiring this parser into \texttt{ingest\_top}, grab the reference frame hex and expected parser outputs bundled with Module~2 (\texttt{sim/reference\_frame.hex} and \texttt{sim/reference\_frame\_expected.txt}). Feed them through your parser in simulation. If the reference frame fails to parse, the bug is local to this handout, not a Module~1 spec drift issue. This pre-check eliminates false-negative debugging against Module~1 firmware.
\end{tcolorbox}

\section{Procedure}

\subsection{Step 1 --- Define the parser states}
Create \texttt{rtl/pkt\_parser.vhd}. States: \texttt{HUNT}, \texttt{LEN}, \texttt{META}, \texttt{PAYLOAD}, \texttt{CRC}, \texttt{VALID}, \texttt{REJECT}.
\begin{itemize}[leftmargin=2em]
	\item \texttt{HUNT}: pop bytes from the \FIFO{} until one equals \texttt{0x7E}.
	\item \texttt{LEN}: consume the length byte; verify it equals \texttt{0x29} (41 decimal). If not, go to \texttt{REJECT}.
	\item \texttt{META}: capture 8 bytes into an 8-byte metadata register array.
	\item \texttt{PAYLOAD}: capture 32 bytes into a 32-byte payload register array.
	\item \texttt{CRC}: capture the \CRC{}-8 byte. Compare against the running \CRC{} computed across bytes 1--41 (computation module in 2-E; stub it with a placeholder for now). Do \emph{not} include the start delimiter (byte 0) in the \CRC{} span.
	\item \texttt{VALID}: one-cycle pulse \texttt{frame\_valid}, return to \texttt{HUNT}.
	\item \texttt{REJECT}: one-cycle pulse \texttt{frame\_reject}, return to \texttt{HUNT}.
\end{itemize}

\labfigure{LH2D_parser_fsm.png}{0.85\textwidth}{Packet parser \FSM{} with the full transition set. Each labeled arc shows the byte or condition that fires the transition.}{fig:lh2d-fsm}

\subsection{Step 2 --- Drive the \FIFO{} correctly}
The parser must assert \texttt{rd\_en} only when \texttt{empty = 0}. Use a one-cycle pipeline delay between the \texttt{rd\_en} pulse and sampling \texttt{rd\_data}: a synchronous \FIFO{} presents new data one clock after \texttt{rd\_en} is high.

\begin{tcolorbox}[warning={Byte-off-by-one}]
	The most common parser bug is sampling \texttt{rd\_data} on the same cycle as the \texttt{rd\_en} pulse. That captures the \textit{previous} byte. Verify the intended timing in a short waveform before adding the full state set.
\end{tcolorbox}

\subsection{Step 3 --- Expose the parsed fields}
Top-level outputs of \texttt{pkt\_parser}:
\begin{itemize}[leftmargin=2em]
	\item \texttt{frame\_valid}, \texttt{frame\_reject} --- one-cycle pulses.
	\item \texttt{metadata(0 to 7)} --- 8 bytes, stable on the \texttt{frame\_valid} cycle.
	\item \texttt{payload(0 to 31)} --- 32 bytes, stable on the \texttt{frame\_valid} cycle.
	\item \texttt{rx\_crc} --- the captured \CRC{}-8 byte (used by 2-E).
\end{itemize}

\subsection{Step 4 --- Parser testbench}
Create \texttt{sim/tb\_pkt\_parser.vhd}. Stimulus:
\begin{enumerate}[leftmargin=2em]
	\item Pre-fill \texttt{byte\_fifo} with a known-good 43-byte frame followed by a corrupted frame (wrong length byte) and a second known-good frame.
	\item Assert that \texttt{frame\_valid} fires exactly twice, \texttt{frame\_reject} fires exactly once, and the captured \texttt{payload} bytes match the known-good reference.
\end{enumerate}

\section{Verification Checklist}
\begin{itemize}[leftmargin=2em]
	\item[$\square$] Parser compiles under \texttt{ingest\_top} with \texttt{byte\_fifo.rd\_en} driven by the parser.
	\item[$\square$] Known-good frame produces a \texttt{frame\_valid} pulse on the correct cycle.
	\item[$\square$] Corrupted length byte produces a \texttt{frame\_reject} pulse and the parser returns to \texttt{HUNT}.
	\item[$\square$] \texttt{payload} register array matches the reference bytes on \texttt{frame\_valid}.
\end{itemize}
