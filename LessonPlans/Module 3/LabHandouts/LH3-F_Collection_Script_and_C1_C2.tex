\labheader{3-F}{Python Collection Script and Campaigns C1 and C2}

\section{Objective}
Implement the host-side Python data collection script that receives the Threat's 43-byte forwarding frames passed through the \FPGA{}'s \mbox{USB-UART} bridge, parses the embedded radio packet fields, attaches ground-truth labels from a campaign configuration file, and writes labeled records to a \mbox{CSV} file. Then execute the first two single-variable campaigns: a \mbox{TX} power sweep (C1) and a spreading factor sweep (C2).

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handout~3-E complete (production bitstream ready, performance envelope documented).
	\item Python~3.10+ on the host PC with \texttt{pyserial $\geq$ 3.5}, \texttt{pandas $\geq$ 2.0}, and \texttt{matplotlib} installed.
	\item \FPGA{} \mbox{USB-UART} passthrough verified: \texttt{ingest\_top.vhd} routes \texttt{uart\_rx\_pin} directly to \texttt{UART\_RXD\_OUT}, so the host PC sees the raw Threat forwarding frames on the Nexys A7's onboard \mbox{USB-UART} bridge at 115200\,8N1.
\end{itemize}

\section{Forwarding Frame Format}
The Threat emits a 43-byte forwarding frame over \mbox{USART3} TX for every intercepted radio packet. The \FPGA{} passes this through unchanged to the host \mbox{USB-UART}. The collection script parses these frames directly --- there is no separate feature-export path on the \FPGA{}.

\begin{table}[h!]
	\centering
	\renewcommand{\arraystretch}{1.3}
	\begin{tabularx}{\textwidth}{l c c X}
		\toprule
		\textbf{Field} & \textbf{Offset} & \textbf{Size} & \textbf{Description} \\
		\midrule
		Start Delimiter & 0 & 1\,B & \texttt{0x7E} \\
		Length & 1 & 1\,B & \texttt{0x29} (41 decimal; bytes covered by \mbox{CRC}) \\
		\texttt{rx\_timestamp\_ms} & 2--5 & 4\,B & Threat \texttt{HAL\_GetTick()} at \texttt{RxDone}, little-endian uint32 \\
		\texttt{rssi\_dbm} & 6 & 1\,B & Signed int8, raw negative dBm (e.g.\ \texttt{0xCE} = $-50$\,dBm) \\
		\texttt{snr\_q025} & 7 & 1\,B & Signed int8, 0.25\,dB units (divide by 4 for dB) \\
		\texttt{freq\_error\_hz} & 8--9 & 2\,B & Signed int16, little-endian, Hz \\
		Radio Payload & 10--41 & 32\,B & Raw intercepted LoRa packet (see embedded fields below) \\
		\mbox{CRC}-8/ITU & 42 & 1\,B & Polynomial \texttt{0x07}, init \texttt{0x00}, covers bytes 1--41 \\
		\bottomrule
	\end{tabularx}
\end{table}

\section{Procedure}

\subsection{Step 1 --- Campaign configuration JSON}
Each campaign run is described by a JSON configuration file that the collection script reads at startup. The configuration provides the ground-truth labels that are attached to each received feature vector.

\begin{lstlisting}[language=Python, caption={Example campaign configuration file \texttt{campaign\_c1\_txpwr\_2.json}.}]
{
    "campaign_id":         "C1",
    "run_id":              "C1_pwr02_2026-04-20T14:00:00Z",
    "sweep_variable":      "tx_power",
    "sweep_value":         2,
    "fixed_sf":            7,
    "fixed_mod":           "lora",
    "fixed_mod_kind":      1,
    "fixed_distance_m":    3.0,
    "num_active_stations": 1,
    "expected_pkt_types":  [0],
    "target_packets":      200,
    "notes":               "TX power sweep, Target A only (DATA only)",
    "description":         "C1 TX Power Sweep -- +2 dBm, SF7, 200 packets, Target A"
}
\end{lstlisting}

\begin{tcolorbox}[infobox={campaign\_id vs run\_id}]
\texttt{campaign\_id} identifies the campaign \emph{family} (C1, C2, \dots, C5). \texttt{run\_id} identifies one specific collection execution (a single invocation of \texttt{collect.py} at one sweep step, with an ISO-8601 timestamp to ensure uniqueness across reruns). Every row written to \mbox{CSV} must carry both. Downstream scripts (LH3-H merge, LH4-A/G splits) key uniqueness and split-leakage checks on \texttt{run\_id}, not on \texttt{campaign\_id}. Subcampaign labels (e.g.\ C5a/C5b/C5c in LH3-G) live in \texttt{campaign\_id}; a subcampaign can contain multiple \texttt{run\_id}s.
\end{tcolorbox}

\begin{tcolorbox}[infobox={Modulation-Kind and Packet-Type Canonical Mappings}]
\texttt{fixed\_mod\_kind} uses the Module~1/Module~2 canonical integer codes: \textbf{1=LoRa, 2=FSK, 3=OOK}. Store this integer directly in the \mbox{CSV} as \texttt{mod\_kind\_label} so that Module~4 ingest does not need a string-to-int conversion table. The feature vector's \texttt{mod\_kind} field at offset~1 uses the same mapping with the low 2 bits cleared (the LH2-F extractor already does this). Do \emph{not} introduce a 0/1 alias.

The packet-type field uses \textbf{0=DATA, 1=ACK}. The \texttt{expected\_pkt\_types} list in the JSON tells the script which \texttt{pkt\_type} values are valid for this run; rows with a \texttt{pkt\_type} outside the list are flagged as cross-domain anomalies but still recorded. (PAUSE frames originate at the Threat and are transmitted over-the-air to silence Targets; they are not received back through the Threat's intercept path, so they never appear in the dataset.)
\end{tcolorbox}

One JSON file exists per sweep step. The \texttt{Code/Tools/CampaignCollect/campaigns/} directory already contains all 11 files: \texttt{dry\_run.json}, four C1 TX-power files (\texttt{campaign\_c1\_txpwr\_2.json} through \texttt{campaign\_c1\_txpwr\_20.json}), and six C2 spreading-factor files (\texttt{campaign\_c2\_sf7.json} through \texttt{campaign\_c2\_sf12.json}). Each file includes a \texttt{description} field that \texttt{collect.py} displays in the selection menu. Output CSVs are written under \texttt{Code/Tools/CampaignCollect/logs/<config\_stem>/}.

\subsection{Step 2 --- Refactor Target firmware for campaign sweeps}
In Lab~2 you wrote \texttt{lora\_init()} with the spreading factor and TX power baked in as raw register bytes. Before running any campaign sweep those values need to change repeatedly, so refactor both Target projects now to expose them as named defines at the top of the file. This is a one-time change; all subsequent steps only require editing two lines.

\begin{enumerate}[leftmargin=2em]
    \item In VS Code, open the \texttt{Nucleo\_RFM95\_TargetA} folder and navigate to \texttt{Core/Src/main.c}. In the \texttt{/* USER CODE BEGIN PD */} block, add the following define block after your existing beacon/role defines:
\begin{lstlisting}[language=C]
/* Campaign-configurable radio parameters.
 * Change LORA_SF and LORA_TX_PWR_DBM between runs.
 * Do not edit the derived macros or any register addresses. */
#define LORA_SF          7
#define LORA_TX_PWR_DBM  14

#define REG_MODEM_CONFIG2_VAL  ((LORA_SF << 4) | 0x04)
#define REG_PA_CONFIG_VAL      (0x80 | ((LORA_TX_PWR_DBM - 2) & 0x0F))
\end{lstlisting}

    \item In \texttt{lora\_init()}, find the two lines where you wrote the spreading factor and TX power registers and replace the raw byte literals with the new macros:
\begin{lstlisting}[language=C]
rfm_write_reg(0x1E, REG_MODEM_CONFIG2_VAL);  /* was 0x74 (SF7) */
rfm_write_reg(0x09, REG_PA_CONFIG_VAL);       /* was 0x8C (+14 dBm) */
\end{lstlisting}

    \item Update the \texttt{printf} at the end of \texttt{lora\_init()} to confirm the active configuration on the debug terminal:
\begin{lstlisting}[language=C]
printf("lora_init done: SF=%d TxPwr=%d dBm\r\n",
       LORA_SF, LORA_TX_PWR_DBM);
\end{lstlisting}

    \item Repeat the identical changes in \texttt{Nucleo\_RFM95\_TargetB/Core/Src/main.c}.
    \item Press \textbf{F7} in each VS Code window to build, then click \textbf{Start Debugging} (or press \textbf{F5}) to flash and run both boards. Confirm both debug terminals print \texttt{lora\_init done: SF=7 TxPwr=14 dBm} before continuing.
\end{enumerate}

\subsection{Step 3 --- Implement the collection script}
Create \texttt{collect.py}. The script must: open the serial port, synchronize on the \texttt{0x7E} start delimiter, read each 43-byte forwarding frame, verify the \mbox{CRC-8/ITU} over bytes~1--41, extract the radio-packet fields from the embedded payload, attach labels from the configuration JSON, and append one row to a \mbox{CSV} file.

\begin{lstlisting}[language=Python, caption={Collection script skeleton (\texttt{collect.py}).}]
import serial
import csv
import json
import sys
from datetime import datetime

BAUD = 115200
FRAME_LEN = 43
START = 0x7E
LENGTH = 0x29  # bytes 1..41 covered by CRC

def crc8_itu(data, poly=0x07, init=0x00):
    crc = init
    for b in data:
        crc ^= b
        for _ in range(8):
            crc = ((crc << 1) ^ poly) & 0xFF if (crc & 0x80) \
                  else (crc << 1) & 0xFF
    return crc

def parse_forwarding_frame(buf43):
    """Unpack a 43-byte Threat forwarding frame into a dict."""
    rx_ts_ms      = int.from_bytes(buf43[2:6],  "little")
    rssi_dbm      = int.from_bytes(buf43[6:7],  "little", signed=True)
    snr_q025      = int.from_bytes(buf43[7:8],  "little", signed=True)
    freq_err_hz   = int.from_bytes(buf43[8:10], "little", signed=True)
    payload       = buf43[10:42]   # 32-byte raw radio packet

    beacon_id     = payload[2]
    seq_num       = int.from_bytes(payload[3:5], "little")
    payload_len   = payload[5]
    mod_code      = payload[6]
    pkt_type      = mod_code & 0x03           # 0=DATA, 1=ACK
    mod_kind      = (mod_code >> 2) & 0x3F    # upper 6 bits
    spread_factor = payload[7]
    tx_ts_ms      = int.from_bytes(payload[9:13], "little")

    return {
        "rx_timestamp_ms":   rx_ts_ms,
        "rssi_dbm":          rssi_dbm,
        "snr_db":            snr_q025 / 4.0,
        "freq_error_hz":     freq_err_hz,
        "beacon_id":         beacon_id,
        "seq_num":           seq_num,
        "payload_len":       payload_len,
        "mod_kind":          mod_kind,
        "spread_factor":     spread_factor,
        "pkt_type":          pkt_type,
        "tx_timestamp_ms":   tx_ts_ms,
    }

def collect(port, config_path, out_csv):
    with open(config_path) as f:
        cfg = json.load(f)
    target = cfg["target_packets"]
    received = 0
    last_rx_ts = None
    with serial.Serial(port, BAUD, timeout=2.0) as ser, \
         open(out_csv, "a", newline="") as csvf:
        writer = csv.DictWriter(csvf, fieldnames=CSV_COLUMNS)
        if csvf.tell() == 0:
            writer.writeheader()
        print(f"Collecting {target} packets for {cfg['campaign_id']} "
              f"sweep={cfg['sweep_value']}...")
        while received < target:
            b = ser.read(1)
            if not b or b[0] != START:
                continue
            length = ser.read(1)
            if not length or length[0] != LENGTH:
                continue   # bad length, resync
            rest = ser.read(FRAME_LEN - 2)
            if len(rest) < FRAME_LEN - 2:
                continue
            buf = bytes([START, LENGTH]) + rest
            expected_crc = crc8_itu(buf[1:42])
            if expected_crc != buf[42]:
                print(f"  [WARN] CRC mismatch: got {buf[42]:#04x}, "
                      f"expected {expected_crc:#04x}")
                continue
            fv = parse_forwarding_frame(buf)
            inter_arrival_ms = (fv["rx_timestamp_ms"] - last_rx_ts) \
                               if last_rx_ts is not None else 0
            last_rx_ts = fv["rx_timestamp_ms"]
            # Mismatch check between self-reported and config labels
            if fv["spread_factor"] != cfg["fixed_sf"]:
                print(f"  [WARN] SF mismatch: reported {fv['spread_factor']}, "
                      f"config {cfg['fixed_sf']}")
            if fv["pkt_type"] not in cfg.get("expected_pkt_types", [0,1,2]):
                print(f"  [WARN] unexpected pkt_type={fv['pkt_type']} "
                      f"(campaign expected {cfg.get('expected_pkt_types')})")
            row = {**fv,
                   "inter_arrival_ms":  inter_arrival_ms,
                   "campaign_id":       cfg["campaign_id"],
                   "run_id":            cfg["run_id"],
                   "timestamp_host":    datetime.utcnow().isoformat(),
                   "distance_m":        cfg["fixed_distance_m"],
                   "label_sf":          cfg["fixed_sf"],
                   "label_mod":         cfg["fixed_mod"],
                   "mod_kind_label":    cfg["fixed_mod_kind"],
                   "label_beacon":      fv["beacon_id"],
                   "label_pkt":         fv["pkt_type"],
                   }
            writer.writerow(row)
            received += 1
            if received % 50 == 0:
                print(f"  {received}/{target} packets collected.")
    print(f"Done. {received} packets written to {out_csv}.")

CSV_COLUMNS = [
    "campaign_id", "run_id", "timestamp_host",
    "rx_timestamp_ms", "tx_timestamp_ms", "inter_arrival_ms",
    "beacon_id", "seq_num", "pkt_type", "mod_kind", "spread_factor",
    "rssi_dbm", "snr_db", "freq_error_hz", "payload_len",
    "distance_m", "label_sf", "label_mod", "mod_kind_label",
    "label_beacon", "label_pkt",
]

def _discover_configs():
    from pathlib import Path
    campaigns_dir = Path(__file__).parent / "campaigns"
    entries = []
    for p in sorted(campaigns_dir.glob("*.json")):
        try:
            with open(p) as f:
                cfg = json.load(f)
            entries.append((p, cfg.get("description",
                                       cfg.get("notes", p.stem))))
        except Exception:
            pass
    return entries

def _menu():
    configs = _discover_configs()
    if not configs:
        print("No JSON config files found in script directory.")
        sys.exit(1)
    print("\n=== Campaign Collect ===")
    print("Available configurations:\n")
    for i, (path, desc) in enumerate(configs, 1):
        print(f"  {i:2d}.  {desc}")
        print(f"       ({path.name})")
    print()
    while True:
        try:
            idx = int(input("Select config number: ").strip()) - 1
            if 0 <= idx < len(configs):
                config_path = configs[idx][0]
                break
            print(f"  Enter a number between 1 and {len(configs)}.")
        except (ValueError, EOFError):
            print("  Invalid input.")
    port = input("COM port (e.g. COM3): ").strip()
    if not port:
        print("No port entered.")
        sys.exit(1)
    default_csv = f"results_{config_path.stem}.csv"
    out_csv = input(f"Output CSV [{default_csv}]: ").strip() \
              or default_csv
    print()
    collect(port, str(config_path), out_csv)

if __name__ == "__main__":
    _menu()
\end{lstlisting}

\begin{tcolorbox}[warning={Label Integrity}]
	The \texttt{label\_*} columns are derived from the campaign configuration (what you set the originating station to), \emph{not} from the on-air fields (what the station self-reports). The self-reported fields (\texttt{spread\_factor}, \texttt{mod\_kind}, \texttt{pkt\_type}) are also recorded for cross-checking. When they disagree, the script logs a warning but keeps the record --- a disagreement indicates a firmware configuration error that is itself informative. Never discard mismatched records silently. \texttt{label\_pkt} is set from the parsed \texttt{pkt\_type} (the canonical authority for this label) rather than from the campaign config, because a single C5 run can produce DATA, ACK, and PAUSE rows whose ground-truth packet type can only be known after the row is captured.
\end{tcolorbox}

         f"results_campaign_c2_sf{sf}.csv"

\end{enumerate}
