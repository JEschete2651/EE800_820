\labheader{3-F}{Python Collection Script and Campaigns C1 and C2}

\section{Objective}
Implement the host-side Python data collection script that receives feature vectors from the \FPGA{} over \mbox{USB-UART}, attaches ground-truth labels from a campaign configuration file, and writes labeled records to a \mbox{CSV} file. Then execute the first two single-variable campaigns: a \mbox{TX} power sweep (C1) and a spreading factor sweep (C2).

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handout~3-E complete (production bitstream ready, performance envelope documented).
	\item Python~3.10+ on the host PC with \texttt{pyserial $\geq$ 3.5}, \texttt{pandas $\geq$ 2.0}, and \texttt{matplotlib} installed.
	\item Module~2 LH2-F feature export \mbox{UART} TX path verified: the \FPGA{} transmits 35-byte export frames on the \mbox{USB-UART} bridge when \texttt{frame\_valid} fires.
\end{itemize}

\section{Feature Export Frame Format}
The \FPGA{} exports each validated feature vector over the Nexys A7's \mbox{USB-UART} (115200\,8N1) using the 35-byte frame defined in Module~2 Week~3:

\begin{table}[h!]
	\centering
	\renewcommand{\arraystretch}{1.3}
	\begin{tabularx}{\textwidth}{l c c X}
		\toprule
		\textbf{Field} & \textbf{Offset} & \textbf{Size} & \textbf{Description} \\
		\midrule
		Start Marker & 0 & 1\,B & \texttt{0xFE} (distinct from \texttt{0x7E} receiver delimiter) \\
		Feature Vector & 1 & 32\,B & Complete 32-byte feature vector (Module~2 Table~5 format) \\
		\mbox{CRC}-8 & 33 & 1\,B & \mbox{CRC}-8 over bytes 1--32 (transport integrity) \\
		End Marker & 34 & 1\,B & \texttt{0xFF} \\
		\bottomrule
	\end{tabularx}
\end{table}

\section{Procedure}

\subsection{Step 1 --- Campaign configuration JSON}
Each campaign run is described by a JSON configuration file that the collection script reads at startup. The configuration provides the ground-truth labels that are attached to each received feature vector.

\begin{lstlisting}[language=Python, caption={Example campaign configuration file \texttt{campaign\_c1\_txpwr\_2.json}.}]
{
    "campaign_id":    "C1",
    "run_id":         "C1_pwr02_2026-04-20T14:00:00Z",
    "sweep_variable": "tx_power",
    "sweep_value":    2,
    "fixed_sf":       7,
    "fixed_mod":      "lora",
    "fixed_mod_code": 1,
    "fixed_distance_m": 3.0,
    "num_beacons":    1,
    "target_packets": 200,
    "notes":          "TX power sweep, first level (+2 dBm)"
}
\end{lstlisting}

\begin{tcolorbox}[infobox={campaign\_id vs run\_id}]
\texttt{campaign\_id} identifies the campaign \emph{family} (C1, C2, \dots, C5). \texttt{run\_id} identifies one specific collection execution (a single invocation of \texttt{collect.py} at one sweep step, with an ISO-8601 timestamp to ensure uniqueness across reruns). Every row written to \mbox{CSV} must carry both. Downstream scripts (LH3-H merge, LH4-A/G splits) key uniqueness and split-leakage checks on \texttt{run\_id}, not on \texttt{campaign\_id}. Subcampaign labels (e.g.\ C5a/C5b/C5c in LH3-G) live in \texttt{campaign\_id}; a subcampaign can contain multiple \texttt{run\_id}s.
\end{tcolorbox}

\begin{tcolorbox}[infobox={Modulation-Code Canonical Mapping}]
\texttt{fixed\_mod\_code} uses the Module~1/Module~2 canonical integer codes: \textbf{1=LoRa, 2=FSK, 3=OOK}. Store this integer directly in the \mbox{CSV} as \texttt{mod\_code\_label} so that Module~4 ingest does not need a string-to-int conversion table. The self-reported \texttt{mod\_code} feature vector byte uses the same mapping. Do \emph{not} introduce a 0/1 alias.
\end{tcolorbox}

Create one JSON file per sweep step. For Campaign~C1 (four TX power levels) you will have four files: \texttt{campaign\_c1\_txpwr\_2.json}, \texttt{campaign\_c1\_txpwr\_8.json}, \texttt{campaign\_c1\_txpwr\_14.json}, and \texttt{campaign\_c1\_txpwr\_20.json}.

\subsection{Step 2 --- Implement the collection script}
Create \texttt{collect.py}. The script must: open the serial port, synchronize on the \texttt{0xFE} start marker, parse each 35-byte export frame, verify the transport \mbox{CRC}-8, extract all feature vector fields, attach labels from the configuration JSON, and append one row to a \mbox{CSV} file.

\begin{lstlisting}[language=Python, caption={Collection script skeleton (\texttt{collect.py}).}]
import serial
import csv
import json
import sys
import time
from datetime import datetime

BAUD = 115200
FRAME_LEN = 35
START = 0xFE
END   = 0xFF

def crc8(data, poly=0x07, init=0x00):
    crc = init
    for b in data:
        crc ^= b
        for _ in range(8):
            crc = ((crc << 1) ^ poly) & 0xFF if (crc & 0x80) \
                  else (crc << 1) & 0xFF
    return crc

def parse_feature_vector(raw32):
    """Unpack the 32-byte feature vector into a dict (Module 2 Table 5)."""
    return {
        "beacon_id":        raw32[0],
        "mod_code":         raw32[1],
        "spread_factor":    raw32[2],
        # byte 3 reserved
        "rssi":             int.from_bytes(raw32[4:6], "big", signed=True),
        "snr":              int.from_bytes(raw32[6:8], "big", signed=True),
        "obs_count":        int.from_bytes(raw32[8:12], "big"),
        "rx_timestamp":     int.from_bytes(raw32[12:16], "big"),
        "inter_arrival_ms": int.from_bytes(raw32[16:20], "big"),
        "rssi_delta":       int.from_bytes(raw32[20:24], "big", signed=True),
        "priority_score":   int.from_bytes(raw32[24:28], "big"),
        # bytes 28-31 reserved
    }

def collect(port, config_path, out_csv):
    with open(config_path) as f:
        cfg = json.load(f)
    target = cfg["target_packets"]
    received = 0
    with serial.Serial(port, BAUD, timeout=2.0) as ser, \
         open(out_csv, "a", newline="") as csvf:
        writer = csv.DictWriter(csvf, fieldnames=CSV_COLUMNS)
        if csvf.tell() == 0:
            writer.writeheader()
        print(f"Collecting {target} packets for {cfg['campaign_id']} "
              f"sweep={cfg['sweep_value']}...")
        while received < target:
            b = ser.read(1)
            if not b or b[0] != START:
                continue
            frame = ser.read(FRAME_LEN - 1)
            if len(frame) < FRAME_LEN - 1:
                continue
            if frame[-1] != END:
                continue
            raw32 = frame[:32]
            expected_crc = crc8(raw32)
            if expected_crc != frame[32]:
                print(f"  [WARN] CRC mismatch: got {frame[32]:#04x}, "
                      f"expected {expected_crc:#04x}")
                continue
            fv = parse_feature_vector(raw32)
            # Mismatch check between self-reported and config labels
            if fv["spread_factor"] != cfg["fixed_sf"]:
                print(f"  [WARN] SF mismatch: reported {fv['spread_factor']}, "
                      f"config {cfg['fixed_sf']}")
            row = {**fv,
                   "campaign_id":       cfg["campaign_id"],
                   "run_id":            cfg["run_id"],
                   "timestamp_host":    datetime.utcnow().isoformat(),
                   "distance_m":        cfg["fixed_distance_m"],
                   "label_sf":          cfg["fixed_sf"],
                   "label_mod":         cfg["fixed_mod"],
                   "mod_code_label":    cfg["fixed_mod_code"],
                   "label_beacon":      fv["beacon_id"],
                   }
            writer.writerow(row)
            received += 1
            if received % 50 == 0:
                print(f"  {received}/{target} packets collected.")
    print(f"Done. {received} packets written to {out_csv}.")

CSV_COLUMNS = [
    "campaign_id", "run_id", "timestamp_host",
    "beacon_id", "mod_code", "spread_factor",
    "rssi", "snr", "obs_count", "rx_timestamp",
    "inter_arrival_ms", "rssi_delta", "priority_score",
    "distance_m", "label_sf", "label_mod", "mod_code_label",
    "label_beacon",
]

if __name__ == "__main__":
    if len(sys.argv) != 4:
        print("Usage: python collect.py <COM_PORT> <config.json> <output.csv>")
        sys.exit(1)
    collect(sys.argv[1], sys.argv[2], sys.argv[3])
\end{lstlisting}

\begin{tcolorbox}[warning={Label Integrity}]
	The \texttt{label\_*} columns are derived from the campaign configuration (what you set the beacon to), \emph{not} from the feature vector fields (what the beacon self-reports). The self-reported fields (\texttt{spread\_factor}, \texttt{mod\_code}) are also recorded for cross-checking. When they disagree, the script logs a warning but keeps the record --- a disagreement indicates a firmware configuration error that is itself informative. Never discard mismatched records silently.
\end{tcolorbox}

\subsection{Step 3 --- Dry run (10-packet validation)}
\begin{enumerate}[leftmargin=2em]
	\item Configure Beacon~A to SF7, +14~dBm, 2-second interval.
	\item Create a test config \texttt{dry\_run.json} with \texttt{target\_packets: 10}.
	\item Run: \texttt{python collect.py COM3 dry\_run.json dry\_run.csv} (substitute your actual COM port).
	\item Open \texttt{dry\_run.csv} and verify: 10 rows, correct column names, \texttt{beacon\_id = 0x01} (or your configured ID), \texttt{spread\_factor = 7}, no \mbox{CRC} warnings in the terminal output.
	\item If the script hangs at startup, the \FPGA{} export path is not running. Program the production bitstream and confirm the Nexys A7 \mbox{USB-UART} port is enumerating on the host PC.
\end{enumerate}

\subsection{Step 4 --- Campaign C1: TX power sweep}
\begin{enumerate}[leftmargin=2em]
	\item Configure Beacon~A: SF7, LoRa, 3~m from the receiver, 2-second fixed interval. Beacon~B powered off.
	\item Iterate through the four TX power levels: +2, +8, +14, +20~dBm. For each level:
	\begin{itemize}[leftmargin=1.2em]
		\item Update the TX power in the beacon firmware and reflash. Confirm the debug \UART{} shows the new power level.
		\item Run \texttt{collect.py} with the appropriate config JSON. Collect 200 packets.
		\item Wait for the script to report ``Done.'' Do not interrupt mid-run.
	\end{itemize}
	\item The four runs produce four separate CSV files. Name them \texttt{c1\_pwr02.csv}, \texttt{c1\_pwr08.csv}, \texttt{c1\_pwr14.csv}, \texttt{c1\_pwr20.csv}.
\end{enumerate}

\subsection{Step 5 --- Validate Campaign C1}
\begin{enumerate}[leftmargin=2em]
	\item Concatenate the four CSV files and compute summary statistics (in Python with \texttt{pandas}):
	\begin{lstlisting}[language=Python, caption={Campaign C1 validation.}]
import pandas as pd
df = pd.concat([pd.read_csv(f) for f in
    ["c1_pwr02.csv","c1_pwr08.csv","c1_pwr14.csv","c1_pwr20.csv"]])
print(f"Total rows: {len(df)}")  # expect 800
print(df.groupby("campaign_id")["rssi"].agg(["mean","std","min","max"]))
	\end{lstlisting}
	\item Verify: 800 total rows; mean \mbox{RSSI} increases (becomes less negative) as \mbox{TX} power increases; \mbox{SNR} positive at all power levels at 3~m distance.
	\item Plot mean \mbox{RSSI} vs.\ \mbox{TX} power (scatter with error bars of $\pm 1 \sigma$). The relationship should be monotonically increasing.
\end{enumerate}

\subsection{Step 6 --- Campaign C2: Spreading factor sweep}
\begin{enumerate}[leftmargin=2em]
	\item Keep Beacon~A at +14~dBm, LoRa, 3~m. Fix the transmission interval at 2 seconds (independent of SF --- the beacon will miss some interval ticks at high SF due to long on-air time, which is expected).
	\item Iterate through SF7, SF8, SF9, SF10, SF11, SF12. For each SF:
	\begin{itemize}[leftmargin=1.2em]
		\item Reflash the beacon firmware with the new spreading factor.
		\item Run \texttt{collect.py} with the corresponding config JSON. Collect 200 packets.
	\end{itemize}
	\item At SF12, each packet takes $\approx$1.5 seconds on-air. The 2-second interval will produce packets at roughly 0.5--0.67~fps. Collecting 200 packets at SF12 takes approximately 5--7 minutes.
\end{enumerate}

\subsection{Step 7 --- Validate Campaign C2}
\begin{enumerate}[leftmargin=2em]
	\item Concatenate all six SF CSV files.
	\item Verify 1200 total rows and that \texttt{label\_sf} cycles through values 7--12.
	\item Compute \texttt{inter\_arrival\_ms} statistics per SF. The median inter-arrival time should increase with SF (each additional spreading factor roughly doubles the on-air time). Generate a box plot of \texttt{inter\_arrival\_ms} by \texttt{label\_sf}.
\end{enumerate}

\section{Verification Checklist}
\begin{itemize}[leftmargin=2em]
	\item[$\square$] Collection script handles \mbox{CRC} mismatches gracefully (logs warning, skips row, continues).
	\item[$\square$] Dry run: 10 rows, correct schema, no errors.
	\item[$\square$] C1: 800 total rows; mean \mbox{RSSI} monotonically increases with \mbox{TX} power.
	\item[$\square$] C2: 1200 total rows; inter-arrival time increases with spreading factor.
	\item[$\square$] No sequence number gaps larger than 3 within a single campaign run (isolated gaps are acceptable; a run of gaps indicates a collection script crash or FPGA reset).
\end{itemize}
