\subsection{Step 1 --- Merging script}
Create \texttt{merge\_dataset.py}. The script must concatenate all campaign CSVs, verify schema consistency, remove duplicates, flag label mismatches, and add a stratified train/val/test split column.

\begin{lstlisting}[language=Python, caption={Dataset merging script (\texttt{merge\_dataset.py}).}]
import pandas as pd
import numpy as np
from pathlib import Path
import hashlib

CAMPAIGN_FILES = [
    "c1_pwr02.csv", "c1_pwr08.csv", "c1_pwr14.csv", "c1_pwr20.csv",
    "c2_sf07.csv", "c2_sf08.csv", "c2_sf09.csv", "c2_sf10.csv",
    "c4_1m.csv", "c4_3m.csv", "c4_5m.csv", "c4_8m.csv",
    "c5a.csv", "c5b.csv",
]

EXPECTED_COLS = [
    "campaign_id", "run_id", "timestamp_host",
    "beacon_id", "mod_kind", "spread_factor", "pkt_type",
    "rssi", "snr", "obs_count", "rx_timestamp",
    "inter_arrival_ms", "rssi_delta", "priority_score",
    "distance_m", "label_sf", "label_mod", "mod_kind_label",
    "label_beacon",
]

# Canonical modulation kinds (Module 1/2 spec): 1=LoRa, 2=FSK, 3=OOK.
MOD_LABEL_TO_KIND = {"lora": 1, "fsk": 2, "ook": 3}

# Canonical packet types in this dataset: 0=DATA, 1=ACK.
PKT_TYPE_DATA, PKT_TYPE_ACK = 0, 1

def stratified_split(df, label_col, fracs=(0.70, 0.15, 0.15), seed=42):
    """Assign split at the run_id level, stratified by label_col.

    Keying on run_id (not campaign_id) prevents leakage when a single
    campaign family contains multiple collection runs.
    """
    rng = np.random.default_rng(seed=seed)
    splits = []
    for label_val, grp in df.groupby(label_col):
        runs = grp["run_id"].unique()
        n = len(runs)
        n_train = max(1, round(n * fracs[0]))
        n_val   = max(1, round(n * fracs[1]))
        shuffled = rng.permutation(runs)
        train_r = set(shuffled[:n_train])
        val_r   = set(shuffled[n_train:n_train + n_val])
        for idx, row in grp.iterrows():
            if row["run_id"] in train_r:
                splits.append((idx, "train"))
            elif row["run_id"] in val_r:
                splits.append((idx, "val"))
            else:
                splits.append((idx, "test"))
    split_map = {idx: s for idx, s in splits}
    return df.index.map(split_map)

dfs = []
for fname in CAMPAIGN_FILES:
    p = Path(fname)
    if not p.exists():
        print(f"[WARN] Missing: {fname}")
        continue
    df = pd.read_csv(p)
    # Schema check
    missing = set(EXPECTED_COLS) - set(df.columns)
    if missing:
        raise ValueError(f"{fname} missing columns: {missing}")
    dfs.append(df)

merged = pd.concat(dfs, ignore_index=True)
print(f"Total rows before dedup: {len(merged)}")

# Dedup at (run_id, beacon_id, obs_count, pkt_type) granularity. run_id
# is required; pkt_type is required because a Beacon~ID can produce both
# DATA and ACK rows in the same run (Target~B in C5b) so dedup at
# (run_id, beacon_id, obs_count) alone would collapse legitimate distinct
# rows.
merged = merged.drop_duplicates(
    subset=["run_id", "beacon_id", "obs_count", "pkt_type"])
print(f"Total rows after dedup:  {len(merged)}")

# Flag label mismatches (self-reported vs. config) using canonical
# Module 1/2 integer mod kinds: 1=LoRa, 2=FSK, 3=OOK. mod_kind_label is
# written directly from the campaign config by LH3-F collect.py. Note
# that for ACK rows (pkt_type=1) the responding Target's spread_factor
# self-report is unaffected by the campaign config's fixed_sf, so we
# only check the mismatch for DATA rows (Target~A is the originator
# whose config we control).
data_mask = merged["pkt_type"] == PKT_TYPE_DATA
merged["label_mismatch"] = 0
merged.loc[data_mask, "label_mismatch"] = (
    (merged.loc[data_mask, "spread_factor"] != merged.loc[data_mask, "label_sf"]) |
    (merged.loc[data_mask, "mod_kind"] != merged.loc[data_mask, "mod_kind_label"])
).astype(int)
n_mismatch = merged["label_mismatch"].sum()
print(f"Label mismatches (DATA only): {n_mismatch} "
      f"({100*n_mismatch/data_mask.sum():.1f}% of DATA rows)")

# Tag operating regime: clean (C1, C2, C4 single-station) vs. contended (C5)
merged["regime"] = merged["campaign_id"].apply(
    lambda c: "contended" if c.startswith("C5") else "clean")
print(merged["regime"].value_counts())

# Stratified split by SF label (primary classification target), keyed on run_id
merged["split"] = stratified_split(merged, "label_sf")

merged.to_csv("merged_dataset.csv", index=False)
print(f"Saved: merged_dataset.csv  ({len(merged)} rows)")
print(merged["split"].value_counts())
\end{lstlisting}

\begin{tcolorbox}[warning={Stratification at the run\_id Level}]
	Split at the \emph{run\_id} level, not at \texttt{campaign\_id} or at the individual-sample level. Samples inside one run share channel conditions and hardware state; splitting samples randomly leaks channel context into the test set. Splitting on \texttt{campaign\_id} alone is also unsafe: one campaign family can contain several distinct collection runs. Keying on \texttt{run\_id} (one value per invocation of \texttt{collect.py}) is the only way to keep splits genuinely independent.
\end{tcolorbox}

\begin{tcolorbox}[infobox={Canonical Modulation Kind and Packet Type Mappings}]
Modulation kind is represented throughout Modules~1--5 using the integer codes \textbf{1=LoRa, 2=FSK, 3=OOK} (Module~1 Table~4). The \texttt{mod\_kind} feature vector byte, the \texttt{mod\_kind\_label} config column, and any Module~4 integer checks use this mapping. The string alias \texttt{label\_mod} (``lora''/``fsk''/``ook'') is retained only for readability and never used in a numeric check. Do not introduce a 0/1 alias.

Packet type is \textbf{0=DATA, 1=ACK, 2=PAUSE}. \texttt{pkt\_type} is extracted from the low 2 bits of the radio packet's \texttt{Modulation Code} byte by the LH2-F bit-field extractor and is the canonical authority for the \texttt{label\_pkt} ground-truth label.
\end{tcolorbox}

\subsection{Step 4 --- EDA Visualization 2: RSSI vs.\ distance with FSPL overlay (C4)}

import numpy as np

c4 = df[df["campaign_id"] == "C4"]
distances = [1, 3, 5, 8]

             fmt="o", capsize=5, label="Measured mean RSSI")

\subsection{Step 8 --- Operating-condition shift analysis}
Compare at least two features between the clean regime (C1, C2, and C4 single-station, DATA-only) and the contended regime (C5 cooperative-target ping-pong), entirely within the hardware dataset. The \texttt{regime} column added by the merge script identifies the source for each row.

\begin{enumerate}[leftmargin=2em]
	\item \textbf{\mbox{RSSI} shift:} Compute mean and standard deviation of \mbox{RSSI} for equivalent baseline conditions (SF7, +14~dBm, $\sim$3\,m), comparing C1's +14~dBm rows against the DATA rows of C5b. Half-duplex shadowing during ACK exchanges produces correlated dropouts that change \mbox{RSSI} variance even though the channel is physically the same.
	\item \textbf{Inter-arrival time shift:} Compare inter-arrival time histograms between the clean regime (tightly clustered $\sim$2\,s) and the contended regime (multi-modal: $\sim$1\,s DATA cadence plus ACK round-trip).
	\item For each comparison, state: the direction of the shift (e.g., ``contended-regime \mbox{RSSI} std is 1.5$\times$ the clean-regime value''), the likely physical cause (half-duplex shadowing, ACK collisions), and the implication for Module~4 (\emph{``a classifier trained only on the clean regime may misclassify ACK rows as DATA when deployed on contended traffic, leading to lower packet-type accuracy than the clean-only test set suggests''}).
\end{enumerate}

\begin{tcolorbox}[infobox={Why Operating-Condition Shift Matters for Module~4}]
	Module~4 will train classifiers on the hardware dataset collected here and evaluate them on held-out hardware data. The LH4-F experiment partitions the data along the \texttt{regime} axis: train on the clean regime (C1--C4), evaluate on the contended regime (C5), and measure the accuracy drop $\Delta = \mathrm{Acc}(\mathrm{clean}\!\to\!\mathrm{clean}) - \mathrm{Acc}(\mathrm{clean}\!\to\!\mathrm{contended})$. The distribution shift documented in this step provides the empirical explanation for that drop. Without this analysis, the Module~4 operating-condition shift experiment has no interpretive foundation.
\end{tcolorbox}
