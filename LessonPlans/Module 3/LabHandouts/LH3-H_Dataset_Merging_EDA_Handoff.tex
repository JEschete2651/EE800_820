\subsection{Step 1 --- Merging script}
Create \texttt{merge\_dataset.py}. The script must concatenate all campaign CSVs, verify schema consistency, remove duplicates, flag label mismatches, and add a stratified train/val/test split column.

\begin{lstlisting}[language=Python, caption={Dataset merging script (\texttt{merge\_dataset.py}).}]
import pandas as pd
import numpy as np
from pathlib import Path
import hashlib

CAMPAIGN_FILES = [
    "c1_pwr02.csv", "c1_pwr08.csv", "c1_pwr14.csv", "c1_pwr20.csv",
    "c2_sf07.csv", "c2_sf08.csv", "c2_sf09.csv",
    "c2_sf10.csv", "c2_sf11.csv", "c2_sf12.csv",
    "c3_lora.csv", "c3_fsk.csv",
    "c4_1m.csv", "c4_3m.csv", "c4_5m.csv", "c4_8m.csv",
    "c5a.csv", "c5b.csv", "c5c.csv", "c5d.csv", "c5e.csv",
]

EXPECTED_COLS = [
    "campaign_id", "run_id", "timestamp_host",
    "beacon_id", "mod_code", "spread_factor",
    "rssi", "snr", "obs_count", "rx_timestamp",
    "inter_arrival_ms", "rssi_delta", "priority_score",
    "distance_m", "label_sf", "label_mod", "mod_code_label",
    "label_beacon",
]

# Canonical modulation codes (Module 1/2 spec): 1=LoRa, 2=FSK, 3=OOK.
MOD_LABEL_TO_CODE = {"lora": 1, "fsk": 2, "ook": 3}

def stratified_split(df, label_col, fracs=(0.70, 0.15, 0.15), seed=42):
    """Assign split at the run_id level, stratified by label_col.

    Keying on run_id (not campaign_id) prevents leakage when a single
    campaign family contains multiple collection runs.
    """
    rng = np.random.default_rng(seed=seed)
    splits = []
    for label_val, grp in df.groupby(label_col):
        runs = grp["run_id"].unique()
        n = len(runs)
        n_train = max(1, round(n * fracs[0]))
        n_val   = max(1, round(n * fracs[1]))
        shuffled = rng.permutation(runs)
        train_r = set(shuffled[:n_train])
        val_r   = set(shuffled[n_train:n_train + n_val])
        for idx, row in grp.iterrows():
            if row["run_id"] in train_r:
                splits.append((idx, "train"))
            elif row["run_id"] in val_r:
                splits.append((idx, "val"))
            else:
                splits.append((idx, "test"))
    split_map = {idx: s for idx, s in splits}
    return df.index.map(split_map)

dfs = []
for fname in CAMPAIGN_FILES:
    p = Path(fname)
    if not p.exists():
        print(f"[WARN] Missing: {fname}")
        continue
    df = pd.read_csv(p)
    # Schema check
    missing = set(EXPECTED_COLS) - set(df.columns)
    if missing:
        raise ValueError(f"{fname} missing columns: {missing}")
    dfs.append(df)

merged = pd.concat(dfs, ignore_index=True)
print(f"Total rows before dedup: {len(merged)}")

# Dedup at (run_id, beacon_id, obs_count) granularity. run_id is required;
# deduping on campaign_id alone would collapse distinct runs of the same
# campaign family into one.
merged = merged.drop_duplicates(
    subset=["run_id", "beacon_id", "obs_count"])
print(f"Total rows after dedup:  {len(merged)}")

# Flag label mismatches (self-reported vs. config) using canonical
# Module 1/2 integer mod codes: 1=LoRa, 2=FSK, 3=OOK. mod_code_label is
# written directly from the campaign config by LH3-F collect.py.
merged["label_mismatch"] = (
    (merged["spread_factor"] != merged["label_sf"]) |
    (merged["mod_code"] != merged["mod_code_label"])
).astype(int)
n_mismatch = merged["label_mismatch"].sum()
print(f"Label mismatches: {n_mismatch} ({100*n_mismatch/len(merged):.1f}%)")

# Stratified split by SF label (primary classification target), keyed on run_id
merged["split"] = stratified_split(merged, "label_sf")

merged.to_csv("merged_dataset.csv", index=False)
print(f"Saved: merged_dataset.csv  ({len(merged)} rows)")
print(merged["split"].value_counts())
\end{lstlisting}

\begin{tcolorbox}[warning={Stratification at the run\_id Level}]
	Split at the \emph{run\_id} level, not at \texttt{campaign\_id} or at the individual-sample level. Samples inside one run share channel conditions and hardware state; splitting samples randomly leaks channel context into the test set. Splitting on \texttt{campaign\_id} alone is also unsafe: one campaign family can contain several distinct collection runs. Keying on \texttt{run\_id} (one value per invocation of \texttt{collect.py}) is the only way to keep splits genuinely independent.
\end{tcolorbox}

\begin{tcolorbox}[infobox={Canonical Modulation Code Mapping}]
Modulation is represented throughout Modules~1--5 using the integer codes \textbf{1=LoRa, 2=FSK, 3=OOK} (Module~1 Table~4). The \texttt{mod\_code} feature vector byte, the \texttt{mod\_code\_label} config column, and any Module~4 integer checks use this mapping. The string alias \texttt{label\_mod} (``lora''/``fsk''/``ook'') is retained only for readability and never used in a numeric check. Do not introduce a 0/1 alias.
\end{tcolorbox}

\subsection{Step 4 --- EDA Visualization 2: RSSI vs.\ distance with FSPL overlay (C4)}
\begin{lstlisting}[language=Python]
import numpy as np

c4 = df[df["campaign_id"] == "C4"]
distances = [1, 3, 5, 8]
means = c4.groupby("distance_m")["rssi"].mean()
stds  = c4.groupby("distance_m")["rssi"].std()

# FSPL model: RSSI(d) = P_tx - 20*log10(d) - 20*log10(f_Hz) + 147.55
f_hz  = 915e6
p_tx  = 14  # dBm
d_range = np.linspace(0.5, 10, 200)
fspl = p_tx - 20*np.log10(d_range) - 20*np.log10(f_hz) + 147.55

plt.figure()
plt.errorbar(means.index, means.values, yerr=stds.values,
             fmt="o", capsize=5, label="Measured mean RSSI")
plt.plot(d_range, fspl, "--", label="FSPL model (+14 dBm, 915 MHz)")
plt.xlabel("Distance (m)")
plt.ylabel("RSSI (dBm)")
plt.title("Campaign C4: RSSI vs. Distance")
plt.legend()

\subsection{Step 8 --- Distribution shift analysis}
Compare at least two features between the engagement simulator (from earlier in the project) and the hardware dataset from this module. If simulator output \mbox{CSV} files are available, load them alongside the hardware dataset.

\begin{enumerate}[leftmargin=2em]
	\item \textbf{\mbox{RSSI} shift:} Compute mean and standard deviation of \mbox{RSSI} for equivalent conditions (same distance, same power level). The simulator uses a clean \mbox{FSPL} model; hardware \mbox{RSSI} includes multipath, antenna pattern irregularities, and receiver front-end noise.
	\item \textbf{Inter-arrival time shift:} Compare the inter-arrival time histograms. The simulator's tick-based engine produces near-exact timing; hardware timing includes STM32 interrupt jitter and \UART{} buffering.
	\item For each comparison, state: the direction of the shift (e.g., ``hardware \mbox{RSSI} is 4~dB lower on average''), the likely physical cause, and the implication for Module~4 (\emph{``a classifier trained on simulator data may underestimate the variability of hardware \mbox{RSSI}, leading to overconfident probability estimates''}).
\end{enumerate}

\begin{tcolorbox}[infobox={Why Distribution Shift Matters for Module~4}]
	Module~4 will train classifiers on the hardware dataset collected here and evaluate them on held-out hardware data. But it will also run an experiment: train on simulator data, evaluate on hardware data, and measure the accuracy drop. The distribution shift documented in this step provides the empirical explanation for that drop. Without this analysis, the Module~4 sim-to-real gap experiment has no interpretive foundation.
\end{tcolorbox}
