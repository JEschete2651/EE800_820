\labheader{1-D}{Bidirectional DATA/ACK Ping-Pong Between Two Cooperative Targets}

\section{Objective}
Implement bidirectional half-duplex LoRa communication between two STM32+RFM95W nodes. Target~A transmits DATA packets on a 1\,s schedule; Target~B receives every DATA and immediately replies with an ACK whose sequence number echoes the received DATA. Both nodes log timestamps so that round-trip latency, packet error rate, and \RSSI{}/\SNR{} can be measured. This handout establishes the bidirectional baseline that Lab Handout~1-F (cooperative target firmware with full \FSM{}) and Lab Handout~1-G (interceptor / Threat firmware) build on.

\section{Prerequisites}
\begin{itemize}[leftmargin=2em]
	\item Lab Handouts~1-A, 1-B, 1-C complete for at least \textbf{two} nodes. The two projects \texttt{Nucleo\_RFM95W\_TargetA} and \texttt{Nucleo\_RFM95W\_TargetB} must already exist from LH1-C Step~1, and at least \texttt{TargetA} must have demonstrated \texttt{RegVersion = 0x12} on its Nucleo in LH1-C Step~5.
\end{itemize}

\section{Materials}
\begin{itemize}[leftmargin=2em]
	\item 2\,$\times$ assembled RFM95W breakouts.
	\item 2\,$\times$ Nucleo-L476RG boards + USB cables.
	\item 2\,$\times$ host USB ports (or a powered USB hub).
	\item Breadboards and jumpers for both nodes.
\end{itemize}

\section{Procedure}

\subsection{Step 1 --- Open the pre-existing TargetA and TargetB projects}

The two projects \texttt{Nucleo\_RFM95W\_TargetA} and \texttt{Nucleo\_RFM95W\_TargetB} were created up front in Lab Handout~1-C, Step~1, with identical baseline peripheral configurations (SPI1, USART2, USART3 with DMA, TIM2 1\,ms tick, GPIO labels for \texttt{RFM\_CS}/\texttt{RFM\_RST}/\texttt{RFM\_DIO0}, EXTI on PA8). \textbf{Do not copy, rename, or move these folders} --- moving STM32 project folders breaks the CMake cache, the launch.json paths, and the STM32 extension's project detection. Instead, open each in its own VS Code window and add code in place.

\begin{enumerate}[leftmargin=2em]
	\item Open \texttt{Nucleo\_RFM95W\_TargetA} in VS Code: \textbf{File $\to$ Open Folder} $\to$ select the \texttt{Nucleo\_RFM95W\_TargetA} folder created in LH1-C. The folder must be the root of the VS Code window.
	\item Press \texttt{F7} to confirm the project still builds. (At this point \texttt{TargetA} contains the LH1-C SPI-bring-up code, which will be removed in Step~2 below.)
	\item Open \texttt{Nucleo\_RFM95W\_TargetB} in a \textbf{second VS Code window} so both projects are open simultaneously, each with its own Run-and-Debug panel and Serial Monitor.
	\item Press \texttt{F7} in the second window to confirm \texttt{TargetB} builds. (\texttt{TargetB} is freshly generated from LH1-C with no application code beyond the CubeMX peripheral init, so it will build immediately with no warnings.)
\end{enumerate}

\begin{tcolorbox}[infobox={Why two physically separate projects with hard-coded roles?}]
	The cooperative targets (Target~A and Target~B) implement different firmware paths --- Target~A initiates DATA, Target~B responds with ACK. Two physically separate projects let students keep both VS Code windows open, each pointing at a different Nucleo, and flash each one independently with a single \textbf{Start Debugging} click. In Lab Handout~1-F, both projects gain the full cooperative-target \FSM{} (including the PAUSE listener path) and remain physically separate; the role distinction does not move to a runtime flag.
\end{tcolorbox}

\subsection{Step 2 --- Add the shared LoRa initialization function}

This step applies to \textbf{both} the \texttt{Nucleo\_RFM95W\_TargetA} and \texttt{Nucleo\_RFM95W\_TargetB} projects. Make the same edits in each project's \texttt{main.c}.

Both targets must use identical physical-layer parameters. Table~\ref{tab:lora-params} lists the Module~1 reference settings; these will also be used in Lab Handouts~1-F and 1-G.

\begin{table}[h!]
	\centering
	\caption{Baseline LoRa physical-layer configuration.}
	\label{tab:lora-params}
	\begin{tabular}{l l l}
		\toprule
		\textbf{Parameter} & \textbf{Value} & \textbf{Register / Value} \\
		\midrule
		Carrier frequency & 906.5\,MHz & \texttt{RegFrfMsb=0xE2, Mid=0xA0, Lsb=0x00} \\
		Spreading factor & SF7 & \texttt{RegModemConfig2 = 0x74} \\

\begin{tcolorbox}[warning={Frequency Match is Critical}]
	Both targets \textbf{must} write the same \texttt{RegFrf*} values. A frequency offset of even a few kHz will prevent reception at SF7. Copy the three bytes from Table~\ref{tab:lora-params} exactly --- do not substitute 915.0\,MHz or any other frequency.
\end{tcolorbox}

\begin{enumerate}[leftmargin=2em]
	\item In \texttt{main.c}, locate \texttt{/* USER CODE BEGIN PFP */} and add prototypes for every function you will define in this lab, alongside the existing LH1-C prototypes:
\begin{verbatim}
/* USER CODE BEGIN PFP */
int __io_putchar(int ch);
uint8_t rfm_read_reg(uint8_t addr);
void rfm_write_reg(uint8_t addr, uint8_t val);
uint32_t rfm_frf_to_hz(uint8_t msb, uint8_t mid, uint8_t lsb);
void lora_init(void);
void rfm_set_mode_rx(void);
void rfm_set_mode_standby(void);
void build_packet(uint8_t *buf, uint8_t sender_id, uint16_t seq,
                  uint8_t pkt_type, uint8_t mod_kind);
uint16_t crc16_ccitt(const uint8_t *data, int len);
/* USER CODE END PFP */
\end{verbatim}

	\item Scroll to \texttt{/* USER CODE BEGIN 4 */} near the bottom of \texttt{main.c} and add the \texttt{lora\_init()} definition immediately after the existing helpers from LH1-C:
\begin{verbatim}
void lora_init(void) {
    /* 1. Enter LoRa SLEEP (required before writing FRF registers) */
    rfm_write_reg(0x01, 0x80);
    HAL_Delay(10);

    /* 2. Carrier frequency: 906.5 MHz */
    rfm_write_reg(0x06, 0xE2);  // RegFrfMsb
    rfm_write_reg(0x07, 0xA0);  // RegFrfMid
    rfm_write_reg(0x08, 0x00);  // RegFrfLsb

    /* 3. Modem config: BW=125 kHz, CR=4/5, explicit header */
    rfm_write_reg(0x1D, 0x72);

    /* 4. Modem config: SF7, single packet, CRC on */
    rfm_write_reg(0x1E, 0x74);

    /* 5. Preamble length: 8 symbols */
    rfm_write_reg(0x20, 0x00);
    rfm_write_reg(0x21, 0x08);

    /* 6. Sync word: 0x34 (private LoRa network) */
    rfm_write_reg(0x39, 0x34);

    /* 7. PA: PA_BOOST, +14 dBm */
    rfm_write_reg(0x09, 0x8C);

    /* 8. FIFO base addresses */
    rfm_write_reg(0x0E, 0x80);
    rfm_write_reg(0x0F, 0x00);

    /* 9. Return to STANDBY */
    rfm_write_reg(0x01, 0x81);
    HAL_Delay(10);

    printf("lora_init done\r\n");
}

void rfm_set_mode_standby(void) {
    rfm_write_reg(0x01, 0x81);  // LoRa STANDBY
}

void rfm_set_mode_rx(void) {
    /* Reset FIFO RX pointer and enter RXCONTINUOUS */
    rfm_write_reg(0x0D, rfm_read_reg(0x0F));  // RegFifoAddrPtr = RxBase
    rfm_write_reg(0x12, 0xFF);                 // Clear all IRQ flags
    rfm_write_reg(0x01, 0x85);                 // LoRa RXCONTINUOUS
}
/* USER CODE END 4 */
\end{verbatim}

	\item Add the \texttt{crc16\_ccitt} helper inside the same \texttt{USER CODE BEGIN 4} block. The polynomial is \texttt{0x1021}, init \texttt{0xFFFF}, no reflect, no XOR-out (CCITT-FALSE):
\begin{verbatim}
uint16_t crc16_ccitt(const uint8_t *data, int len) {
    uint16_t crc = 0xFFFF;
    for (int i = 0; i < len; i++) {
        crc ^= ((uint16_t)data[i]) << 8;
        for (int b = 0; b < 8; b++) {
            crc = (crc & 0x8000) ? (uint16_t)((crc << 1) ^ 0x1021)
                                 : (uint16_t)(crc << 1);
        }
    }
    return crc;
}
\end{verbatim}

	\item Add the \texttt{build\_packet} helper that constructs a 32-byte radio packet per the Module~1 packet format (Table~4 in the lesson plan). For LH1-D the payload region is filled with a fixed 17-byte test pattern; LH1-F will replace this with structured per-packet payloads.
\begin{verbatim}
void build_packet(uint8_t *buf, uint8_t sender_id, uint16_t seq,
                  uint8_t pkt_type, uint8_t mod_kind) {
    /* 0-1: Preamble sync 0xAE5A (big-endian) */
    buf[0] = 0xAE; buf[1] = 0x5A;
    /* 2: Sender Beacon ID */
    buf[2] = sender_id;
    /* 3-4: Sequence number (little-endian) */
    buf[3] = (uint8_t)(seq);
    buf[4] = (uint8_t)(seq >> 8);
    /* 5: TX power (dBm) */
    buf[5] = 14;
    /* 6: Modulation Code, packed: bits[7:2]=mod_kind<<2, bits[1:0]=pkt_type */
    buf[6] = (uint8_t)((mod_kind << 2) | (pkt_type & 0x03));
    /* 7: Spreading factor */
    buf[7] = 7;
    /* 8: Bandwidth code (0x00 = 125 kHz) */
    buf[8] = 0x00;
    /* 9-12: Timestamp */
    uint32_t ts = HAL_GetTick();
    buf[9]  = (uint8_t)(ts);
    buf[10] = (uint8_t)(ts >> 8);
    buf[11] = (uint8_t)(ts >> 16);
    buf[12] = (uint8_t)(ts >> 24);
    /* 13-29: Payload (fixed test pattern for LH1-D) */
    for (int i = 13; i < 30; i++) buf[i] = 0xA5;
    /* 30-31: CRC-16/CCITT-FALSE over bytes 0-29, MSB-first */
    uint16_t crc = crc16_ccitt(buf, 30);
    buf[30] = (uint8_t)(crc >> 8);
    buf[31] = (uint8_t)(crc);
}
\end{verbatim}

	\item Locate \texttt{/* USER CODE BEGIN 2 */}. The LH1-C reset sequence and \texttt{RegVersion} read are already there. Add a call to \texttt{lora\_init()} immediately after the \texttt{RegVersion} print, and \textbf{remove or comment out} any LH1-C demo code that follows (the frequency write/read-back and the mode-transition demo). Final state of the block:
\begin{verbatim}
/* USER CODE BEGIN 2 */

/* Reset the RFM95W (from LH1-C) */
HAL_GPIO_WritePin(RFM_RST_GPIO_Port, RFM_RST_Pin, GPIO_PIN_RESET);
HAL_Delay(1);
HAL_GPIO_WritePin(RFM_RST_GPIO_Port, RFM_RST_Pin, GPIO_PIN_SET);
HAL_Delay(10);

/* Sanity check (from LH1-C) */
uint8_t version = rfm_read_reg(0x42);
printf("RegVersion = 0x%02X\r\n", version);

/* Configure the LoRa modem */
lora_init();

/* USER CODE END 2 */
\end{verbatim}

	\item Press \texttt{F7} in each VS Code window to build both projects. Confirm zero errors before continuing.
\end{enumerate}

\subsection{Step 3 --- Add Target~A (DATA initiator + ACK listener) main-loop code}

This step applies \textbf{only} to the \texttt{Nucleo\_RFM95W\_TargetA} project.

\begin{enumerate}[leftmargin=2em]
	\item In \texttt{main.c} of the \texttt{Nucleo\_RFM95W\_TargetA} project, locate \texttt{/* USER CODE BEGIN 2 */} and add a sequence counter and timing variables immediately after the \texttt{lora\_init()} call:
\begin{verbatim}
/* USER CODE BEGIN 2 */
/* ... reset, RegVersion read, lora_init() above ... */

uint16_t seq = 0;
uint32_t next_tx_ms = HAL_GetTick() + 1000;  // first DATA at t=1s
uint16_t acks_received = 0;
uint16_t data_sent = 0;

printf("Target A initiator ready\r\n");

/* USER CODE END 2 */
\end{verbatim}

	\item Locate \texttt{/* USER CODE BEGIN WHILE */} and add the per-iteration TX-then-RX-window code. Target~A transmits a DATA packet, switches into RX, polls for an ACK with a 200\,ms RX-window timeout, then logs the round-trip outcome:
\begin{verbatim}
/* USER CODE BEGIN WHILE */
while (1)
{
    if (HAL_GetTick() >= next_tx_ms) {
        next_tx_ms += 1000;  // schedule next DATA in 1 second

        /* --- Build and transmit DATA --- */
        uint8_t pkt[32];
        build_packet(pkt, BEACON_ID_A, seq,
                     PKT_TYPE_DATA, MOD_KIND_LORA);

        rfm_set_mode_standby();
        /* Point FIFO to TX base address */
        rfm_write_reg(0x0D, rfm_read_reg(0x0E));
        /* Write packet into FIFO */
        for (int i = 0; i < 32; i++) rfm_write_reg(0x00, pkt[i]);
        rfm_write_reg(0x22, 32);  // RegPayloadLength

        uint32_t t_tx_ms = HAL_GetTick();
        rfm_write_reg(0x12, 0xFF);  // Clear IRQ flags
        rfm_write_reg(0x01, 0x83);  // LoRa TX

        /* Wait for TxDone (IRQ flag bit 3) */
        uint8_t irq;
        do {
            irq = rfm_read_reg(0x12);
        } while ((irq & 0x08) == 0);
        rfm_write_reg(0x12, 0xFF);

        data_sent++;

        /* --- Switch to RX and wait up to 200 ms for an ACK --- */
        rfm_set_mode_rx();
        uint32_t rx_window_end = HAL_GetTick() + 200;
        uint8_t got_ack = 0;
        uint32_t t_ack_ms = 0;
        uint16_t ack_seq = 0;
        int16_t ack_rssi = 0;

        while (HAL_GetTick() < rx_window_end) {
            irq = rfm_read_reg(0x12);
            if (irq & 0x40) {  /* RxDone */
                if ((irq & 0x20) == 0) {  /* CRC ok */
                    uint8_t rx_addr = rfm_read_reg(0x10);
                    rfm_write_reg(0x0D, rx_addr);
                    uint8_t rx[32] = {0};
                    for (int i = 0; i < 32; i++) {
                        rx[i] = rfm_read_reg(0x00);
                    }
                    /* Validate sender = B and pkt_type = ACK */
                    uint8_t pkt_type_rx = rx[6] & 0x03;
                    if (rx[2] == BEACON_ID_B
                        && pkt_type_rx == PKT_TYPE_ACK) {
                        got_ack = 1;
                        t_ack_ms = HAL_GetTick();
                        ack_seq = (uint16_t)rx[3]
                                | ((uint16_t)rx[4] << 8);
                        ack_rssi = (int16_t)rfm_read_reg(0x1A) - 157;
                    }
                }
                rfm_write_reg(0x12, 0xFF);
                if (got_ack) break;
            }
        }
        rfm_set_mode_standby();

        /* --- Log CSV: seq, t_tx_ms, t_ack_ms, rtt_ms, rssi_dbm --- */
        if (got_ack) {
            acks_received++;
            uint32_t rtt = t_ack_ms - t_tx_ms;
            printf("CSV,%u,%lu,%lu,%lu,%d\r\n",
                   (unsigned)seq, (unsigned long)t_tx_ms,
                   (unsigned long)t_ack_ms, (unsigned long)rtt,
                   (int)ack_rssi);
        } else {
            printf("CSV,%u,%lu,0,0,0  (no ACK; seq=%u)\r\n",
                   (unsigned)seq, (unsigned long)t_tx_ms,
                   (unsigned)seq);
        }

        if ((data_sent % 10) == 0) {
            printf("# stats: sent=%u acked=%u (%.1f%%)\r\n",
                   data_sent, acks_received,
                   100.0 * acks_received / data_sent);
        }

        seq++;
    }
    /* USER CODE END WHILE */

    /* USER CODE BEGIN 3 */
}
/* USER CODE END 3 */
\end{verbatim}
	The \texttt{USER CODE END WHILE} and \texttt{USER CODE BEGIN 3} markers are already present in the CubeMX-generated file --- do not add or remove them. Your code goes \emph{between} \texttt{USER CODE BEGIN WHILE} and \texttt{USER CODE END WHILE}.

	\item Press \texttt{F7} to build. Confirm zero errors.
\end{enumerate}

\subsection{Step 4 --- Add Target~B (DATA listener + ACK responder) main-loop code}

This step applies \textbf{only} to the \texttt{Nucleo\_RFM95W\_TargetB} project.

\begin{enumerate}[leftmargin=2em]
	\item In \texttt{main.c} of the \texttt{Nucleo\_RFM95W\_TargetB} project, locate \texttt{/* USER CODE BEGIN 2 */} and immediately after \texttt{lora\_init()} put the radio into continuous RX:
\begin{verbatim}
/* USER CODE BEGIN 2 */
/* ... reset, RegVersion read, lora_init() above ... */

rfm_set_mode_rx();
uint16_t data_rx = 0;
uint16_t acks_sent = 0;

printf("Target B responder ready\r\n");

/* USER CODE END 2 */
\end{verbatim}

	\item Locate \texttt{/* USER CODE BEGIN WHILE */} and add the per-iteration RX-then-ACK code:
\begin{verbatim}
/* USER CODE BEGIN WHILE */
while (1)
{
    uint8_t irq = rfm_read_reg(0x12);

    if (irq & 0x40) {  /* RxDone */
        if (irq & 0x20) {  /* CRC error */
            printf("CRC error\r\n");
            rfm_write_reg(0x12, 0xFF);
        } else {
            /* Read packet */
            uint8_t rx_addr = rfm_read_reg(0x10);
            rfm_write_reg(0x0D, rx_addr);
            uint8_t rx[32] = {0};
            for (int i = 0; i < 32; i++) {
                rx[i] = rfm_read_reg(0x00);
            }
            int16_t rssi = (int16_t)rfm_read_reg(0x1A) - 157;
            int8_t  snr_raw = (int8_t)rfm_read_reg(0x19);
            int16_t snr = snr_raw / 4;
            rfm_write_reg(0x12, 0xFF);  /* clear IRQs */

            uint8_t sender = rx[2];
            uint8_t pkt_type_rx = rx[6] & 0x03;
            uint16_t rx_seq = (uint16_t)rx[3]
                            | ((uint16_t)rx[4] << 8);

            if (sender == BEACON_ID_A
                && pkt_type_rx == PKT_TYPE_DATA) {
                data_rx++;
                printf("RX DATA seq=%u rssi=%d snr=%d\r\n",
                       (unsigned)rx_seq, (int)rssi, (int)snr);

                /* --- Build and TX ACK echoing the DATA's seq --- */
                uint8_t ack[32];
                build_packet(ack, BEACON_ID_B, rx_seq,
                             PKT_TYPE_ACK, MOD_KIND_LORA);

                rfm_set_mode_standby();
                rfm_write_reg(0x0D, rfm_read_reg(0x0E));
                for (int i = 0; i < 32; i++) {
                    rfm_write_reg(0x00, ack[i]);
                }
                rfm_write_reg(0x22, 32);
                rfm_write_reg(0x12, 0xFF);
                rfm_write_reg(0x01, 0x83);  /* LoRa TX */

                /* Wait for TxDone */
                do {
                    irq = rfm_read_reg(0x12);
                } while ((irq & 0x08) == 0);
                rfm_write_reg(0x12, 0xFF);

                acks_sent++;

                /* Return to RXCONTINUOUS for next DATA */
                rfm_set_mode_rx();
            }
        }
    }
    /* USER CODE END WHILE */

    /* USER CODE BEGIN 3 */
}
/* USER CODE END 3 */
\end{verbatim}

	\item Press \texttt{F7} to build. Confirm zero errors.
\end{enumerate}

\subsection{Step 6 --- Capture 100 ping-pong pairs and analyze}
\begin{enumerate}[leftmargin=2em]
	\item In the Target~A Serial Monitor, capture at least 100 consecutive \texttt{CSV,...} lines:
	\begin{enumerate}[leftmargin=1.5em]
		\item Right-click in the Serial Monitor output area $\to$ \textbf{Select All} $\to$ \textbf{Copy} (or press \texttt{Ctrl+A} then \texttt{Ctrl+C}).
		\item Paste into a text editor and save as \texttt{LH1D\_pingpong\_log.csv}. Strip the non-CSV header lines (\texttt{RegVersion}, \texttt{lora\_init done}, \texttt{Target A initiator ready}, periodic \texttt{\# stats} lines) so only the \texttt{seq,t\_tx\_ms,t\_ack\_ms,rtt\_ms,rssi\_dbm} rows remain. Add a header row.
	\end{enumerate}
	\item From \texttt{LH1D\_pingpong\_log.csv}, compute:
	\begin{itemize}[leftmargin=1.5em]
		\item \textbf{ACK reception rate}: rows with \texttt{t\_ack\_ms} $\ne$ 0 divided by total rows. Target: $\geq$95\%.
		\item \textbf{Round-trip latency statistics}: mean, standard deviation, minimum, maximum, and 99th-percentile of \texttt{rtt\_ms} across rows where \texttt{t\_ack\_ms} $\ne$ 0. The expected mean is $\sim$60--80\,ms at SF7 with a 32-byte payload (two 50\,ms airtimes plus turnaround).
		\item \textbf{Mean \RSSI{}} and \textbf{standard deviation of \RSSI{}} across all rows with valid ACK.
		\item \textbf{Sequence gaps}: any jump in \texttt{seq} indicates a dropped DATA on Target~B; any \texttt{t\_ack\_ms} $=$ 0 with later sequence numbers continuing indicates a dropped ACK on Target~A.
	\end{itemize}
	\item Compare mean \RSSI{} against the Module~1 link budget estimate for 1\,m indoor separation. For Friis free-space path loss at 906.5\,MHz over 1\,m:
	\[
		L_{\text{fs}} = 20\log_{10}\!\left(\frac{4\pi d f}{c}\right) \approx 31.6\,\text{dB}
	\]
	With +14\,dBm transmit power and $\sim-2$\,dBi antenna efficiency on each end, expected received power $\approx -22$\,dBm. Indoor multipath will shift this; a measured value between $-30$ and $-50$\,dBm at 1\,m is typical. A discrepancy $>$10\,dB warrants an antenna inspection.
\end{enumerate}

\begin{tcolorbox}[warning={Troubleshooting}]
	\begin{itemize}[leftmargin=1.2em]
		\item \textbf{No ACKs received on Target~A}: confirm Target~B's \texttt{Modulation Code} byte for ACK is \texttt{(MOD\_KIND\_LORA $\ll$ 2) | PKT\_TYPE\_ACK} = \texttt{0x05}, not bare \texttt{0x01}. The packed type field is the most common bug introduced in Step~2.
		\item \textbf{Target~A logs all-zero ACKs}: Target~A's RX-window timeout (200\,ms) is too short for the round-trip at the current SF/payload. At SF7 with 32-byte packets, two airtimes total $\sim$100\,ms; widen the window to 300\,ms if Target~B is verifiably transmitting.
		\item \textbf{Target~B prints \texttt{CRC error} on every packet}: verify \texttt{CrcOn} is set (\texttt{RegModemConfig2 = 0x74}) on the Target~A side and that the preamble length (\texttt{0x08}) matches.
		\item \textbf{Sequence gaps on Target~B}: usually transient half-duplex collisions if Target~B failed to switch back into RX before the next Target~A DATA. Confirm that \texttt{rfm\_set\_mode\_rx()} is called at the end of the ACK path before the loop iterates.
		\item \textbf{Very low \RSSI{} ($<-$90\,dBm at 1\,m)}: usually an antenna problem --- re-check the spring antenna solder joint on both modules (LH1-A).
		\item \textbf{Build fails after the LH1-C demo code is removed}: stale \mbox{CMake} cache. Delete the \texttt{build/} folder in the project root and press \texttt{F7} again.
	\end{itemize}
\end{tcolorbox}
